# 表格（表）處理總結

## 基本原理

國中教育會考社會科題本中，表格標籤 `表(N)` 位於表格**上方**，裁切方向為**往下**搜尋表格邊界。此與圖形相反。

## 表格特性

- 大部分表格**沒有**單一向量矩形框架，而是由**個別線條**繪製的儲存格邊框組成
- 表格儲存格文字以文字區塊形式存在，非繪圖元素
- 表格標籤通常置中於表格上方，表格可為右欄（約 x>300）或全幅
- 部分表格可能與圖形共享同一頁面，需避免誤抓圖形框架
- 表格行之間可能有小的繪圖元素（文字底線等）干擾叢集合併

## PDF 編碼陷阱

- **注音符號 `ㄧ` (U+3127)**：部分表格標籤中的 `一` 被 PDF 字型編碼為注音符號而非漢字。需要在此字元出現的所有正則表達式和數字對照表中加入支援。
  ```python
  CN_DIGITS = {'一':1, ..., '\u3127':1}
  regex: [一二三四五六七八九十\u3127\d]+
  ```

## 四階段偵測流程（依優先序）

### 1. 框架偵測（frame）— 表格專屬
- 搜尋方向：**標籤下方**（與圖形相反）
- 匹配條件（較圖形更嚴格）：
  ```
  rb[1] >= ly                         # 框頂必須在標籤 y 位置或之下（不可在標籤上方）
  rb[1] <= ly + 40                    # 框頂距標籤 ≤40pt（緊鄰標籤）
  rb[3] <= ly + 500                   # 框底不超過標籤下方 500pt
  abs((rb[0]+rb[2])/2 - lx) < 100     # 框中心 x 必須在標籤 x 的 ±100pt 內
  ```
- **核心設計原則**：使用 `abs(框中心_x - 標籤_x) < 100` 而非寬鬆的 x 範圍檢查，避免匹配到同一頁面上的圖形框架
- 裁切範圍：`[框左, max(0, 框頂-10), 框右, 框底]`

### 2. 嵌入式圖片偵測（embedded）— 表格專屬
- 極少使用（表格很少是嵌入式圖片）
- 匹配條件與 frame 相同

### 3. 繪圖元素叢集（cluster）— 表格專屬
- **這是表格偵測的核心方法**，因為多數表格使用線條繪製
- 篩選條件（與圖形 cluster 完全不同）：
  ```
  max(r.width, r.height) > 3                              # 支援水平線（h=0）和垂直線（w=0）
  r.x0 > 50                                                # 含表格左側邊框（比圖形的 310 寬很多）
  abs((r.x0 + r.x1) / 2 - label["x"]) < 300                # x 中心必須在標籤 ±300pt 內（含左側行標籤）
  r.y0 >= ly - 10                                          # 標籤上方 10pt 以內
  r.y1 <= ly + 300                                         # 標籤下方 300pt 以內
  ```
- **x 中心接近度過濾** (`abs(x_center - lx) < 300`)：
  - 解決 stray element 問題：頁面上常有與表格同 y 範圍但 x 位置完全不同的繪圖元素（如左欄題目文字的底線、小點），這些元素會打散叢集合併
  - 以標籤 x 位置為中心，±300pt 範圍足以涵蓋全幅表格（含左側行標籤如 x=71 的「甲、乙、丙、丁」），同時排除遠在 x=100-150 的干擾元素
- 叢集合併容忍：
  - y 軸：`b[1] <= cb[3] + 40`（容忍 40pt 間隙，合併被小元素打散的表格行）
  - x 軸：`b[0] <= cb[2] + 30 and b[2] >= cb[0] - 30`（適中的 x 重疊容忍，配合 x 中心過濾後可安全使用）
- 合格叢集：`width > 50, height > 30, len > 5`
- 叢集評分（距離優先，含 x 偏移懲罰）：
  ```
  dist = cb[1] - ly + abs((cb[0]+cb[2])/2 - label["x"]) * 0.5
  score = 5000 - dist * 10 + len(c)
  ```
  - y 距離加權 10 倍，x 距離加權 0.5 倍後再乘 10 倍（即 x 每偏移 1pt = 扣 5 分）
  - 大幅偏好靠近標籤的叢集，防止選到頁面下方或左右偏移的圖形叢集
  - `best_score` 初始化為 `-999999`（允許負分），確保至少有一個叢集被選中
- 僅考慮 `dist < 150` 且在標籤之下的叢集
- 叢集底部上限：`min(cb[3], ly + 200)`（防止叢集合併到不相關內容）
- 裁切範圍：`[叢集左, max(0, 叢集頂-15), 叢集右, 叢集底+5]`

### 4. 啟發式裁切（crop）— 表格專屬
- 適用於前三種方法都找不到的表格
- 下邊界 `fig_bottom`：
  - 預設 `ly + 170`
  - 若下方有標籤且比 `ly + 170` 更近 → 使用該標籤位置 -15pt
- 左邊界：
  - 若標籤 x > `page_w * 0.4`（約 238pt）→ 使用 `x=305`（右欄表格）
  - 否則 → 使用 `x=66`（全幅表格）
- 裁切範圍：`[left_x, ly + 5, page_w - 55, fig_bottom]`

## 關鍵參數比較（圖 vs 表）

| 參數 | 圖形 | 表格 |
|------|------|------|
| 搜尋方向 | 往上（`ly - 500` ~ `ly + 20`） | 往下（`ly` ~ `ly + 500`） |
| cluster 初始元素過濾 | `x0 > 310` | `x0 > 50` + `abs(x_center-lx) < 300` |
| cluster 尺寸過濾 | `w>5 and h>5` | `max(w,h) > 3` |
| cluster 最小元素數 | 10 | 5 |
| cluster 合併 y 容忍 | 10pt | 40pt |
| cluster 合併 x 容忍 | 10pt | 30pt |
| cluster 合格高度 | 50pt | 30pt |
| cluster 評分機制 | 原始（選最大叢集） | 距離優先（5000 - dist×10 + len） |
| crop 預設範圍 | `ly - 200` ~ `ly` | `ly` ~ `ly + 170` |
| crop 左邊界 | 300 或 66 | 305 或 66 |
| frame 匹配關鍵 | 框底靠近標籤 | 框**中心**靠近標籤 + 框頂在標籤之下 |
| frame 右界 | `page_w - 55` | 框原始右界 |
| 小型結果過濾 | cw < 150 捨棄 | 無（表格通常較窄） |
| 同階面積優先 | area > 1.5x 取代 | 同左 |

## 輸出格式
- 檔名：`{year}t{NN:02d}.jpg`（如 `114t01.jpg`）
- JSON 記錄：`web/pic/figures_{year}.json`（同圖形）

## 常見問題與解決方案

| 問題 | 原因 | 解決方案 |
|------|------|---------|
| 表格抓到圖形 | frame 匹配太寬鬆，x 範圍檢查不足以區分 | 使用 x 中心比對 `abs(center-lx)<100`，並要求框頂 `rb[1] >= ly`（不可在標籤上方） |
| 表格左側欄位缺失 | cluster 元素過濾太右（`x0>100` 仍不足以含行標籤 x=71） | 降為 `x0>50` + `abs(x_center-lx)<300` |
| 表格只抓到標頭/部分行 | stray element（左欄底線/小點）打散叢集合併 | 加入 `abs(x_center-lx)<300` 過濾，排除 x 位置不同的干擾元素 |
| cluster 選到下方圖形 | 距離評分未考慮 x 偏移 | 加入 `abs(x_center-lx)*0.5` 到 dist 計算 |
| crop 抓到題目文字 | 下邊界太大或左邊界判定錯誤 | 使用 `ly+170` + 依標籤 x 位置決定左邊界 (`page_w * 0.4` 閾值) |
| crop 抓到左側題目 | 全幅預設太寬，或標籤 x 在頁面中線導致誤判為左欄 | 使用 `page_w * 0.4`（非 `page_w / 2`）作為閾值 |
| 表格誤判為圖形（或相反）| 共用偵測函式導致參數互相污染 | 圖/表偵測函式完全分離，各自獨立的四階段 pipeline |
