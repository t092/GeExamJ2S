# 圖片選項題處理規劃

> 部分題目的選項本身是圖片（非文字），parser 無法擷取文字內容導致 JSON 為空。
> 本文規劃一次裁切整個選項區（含 (A)(B)(C)(D) 標籤 + 圖）作為單張 JPG，
> 前端直接以圖片顯示選項。

---

## 0. 摘要

- **問題**：parser 抽不出圖片選項的文字，JSON 中 `options.A/B/C/D` 為空字串 →
  前端無法顯示。
- **解法**：將該題的整個選項區（含 `(A)...(B)...(C)...(D)...` 與其下的圖）
  一次裁切為單張 JPG，前端改以圖片代替選項列表。
- **存放**：`web/pic/item/{year}{q}i.jpg`（如 `11443i.jpg`、`11529i.jpg`）。
- **不影響**：現有 `web/pic/{year}pNN.jpg`（題幹圖表）命名規則不變。

---

## 1. 全案例分析

### 1.1 受影響題目清單

從三 / 四年 JSON 掃描空選項題，共 8 題：

| 年 | 題號 | 類型 | 頁碼 | 空選項 | 預期檔名 |
|----|------|------|------|--------|---------|
| 112 | 16 | 單題 | 5 | A,B,C,D | `11216i.jpg` |
| 112 | 39 | 單題 | 11 | A,B,C | `11239i.jpg` |
| 113 | 7 | 單題 | 3 | A,B,C,D | `11307i.jpg` |
| 114 | 29 | 單題 | 8 | A,B,C | `11429i.jpg` |
| 114 | 43 | 題組子題 | 12 | A,B,C | `11443i.jpg` |
| 115 | 27 | 單題 | 8 | A,B,C | `11527i.jpg` |
| 115 | 29 | 單題 | 8 | A,B,C,D | `11529i.jpg` |
| 115 | 41 | 單題 | 11 | A,B,C,D | `11541i.jpg` |

> 註：112 Q39、114 Q29、114 Q43、115 Q27 只有 A、B、C 三選項（D 是文字）→
> 裁切時只要涵蓋有圖的 (A)(B)(C)，但要驗證 D 文字是否仍應顯示。
> 详见 §3.3。

### 1.2 為什麼 parser 抽不出來

PyMuPDF `page.get_text()` 只能萃取文字向量，**點陣圖選項**（如手繪地圖、
符號圖示、印章圖形）在文字流中只留下空白。選項標籤 `(A)` `(B)` 雖是文字，
但因夾在圖片中央，其位置難以與文字流排序，且經常與圖片 bbox 重疊
而被 PyMuPDF 以「圖片內文字」跳過。

### 1.3 題目在題本中的版面特徵

依 PDF 視覺版面，這類題目的選項區有兩種排列：

**類型 A — 四圖並排，下方各帶標籤**（如 115 Q29）
```
[圖A]   [圖B]   [圖C]   [圖D]
(A)     (B)     (C)     (D)
```

**類型 B — 三圖並排 + 一個文字選項**（如 112 Q39）
```
[圖A]   [圖B]   [圖C]    (D)文字
(A)     (B)     (C)
```

類型 B 的 D 選項在 parser 中應該仍能抓到文字 → 比 JSON 中的 `options.D`
即可判斷（非空字串）。但前述盤點發現 112 Q39 `options.C` 也是空、D 都為空，
需在實作時逐一核對原題本確認排版後再決定裁切範圍。

## 2. 命名與目錄規則

```
web/pic/
├── 114p01.jpg ~ 114pNN.jpg     # 題幹圖（原有，名稱不變）
├── item/                       # 圖片選項題專用（新增）
│   ├── 11216i.jpg
│   ├── 11239i.jpg
│   ├── 11307i.jpg
│   ├── 11429i.jpg
│   ├── 11443i.jpg
│   ├── 11527i.jpg
│   ├── 11529i.jpg
│   └── 11541i.jpg
└── ...
```

### 2.1 命名規則

- 目錄：`web/pic/item/`
- 檔名：`{year}{q:02d}i.jpg`
  - `year`：3 位年度碼（112/113/114/115...）
  - `q`：題號，2 位補零
  - `i`：image-option 的識別後綴（與 `p` 題幹圖區隔）
- 大小寫：全小寫
- 編碼：JPG（品質 90，與現有 `p` 圖一致）

### 2.2 不變動現有命名

- `web/pic/{year}pNN.jpg`：題幹圖表，沿用舊命名
- `web/pic/item/{year}{q}i.jpg`：選項圖，獨立目錄
- 兩者命名規則互不衝突，前端可依路徑區分。

## 3. 裁切流程規劃

### 3.1 輸入

- PDF 原題本：`{year}年國中教育會考社會科題本.pdf`
- 頁碼渲染圖（已存在）：`assets/{year}/pages/page_NN.png`
- 題目頁碼對應（已盤點，见 §1.1）

### 3.2 裁切框 (bbox) 定位策略

每個題目需要確定「選項區」的 bbox。候選策略（由穩到鬆）：

**Layer 0：題號起點 + 下一題起點**
1. 找題號 `N.` 的文字 bbox 起點（PyMuPDF `page.get_text("dict")` 可取）
2. 找下一題題號 `N+1.` 起點（或題組的下一個題組頭）
3. 兩者之間即為本題範圍
4. 在本題範圍內找 `(A)`、`(D)` 標籤的 y 座標範圍 → 選項區的 y 區間
5. x 區間取頁面有效寬度（去除左右邊界）

**Layer 1：使用 frame rects**
- 沿用 `crop_figures_v3.py` 的 `page.get_drawings()` 找框線 rect
- 選項區通常有外框（border）包住 4 個選項格子
- 找到包含 `(A)` 標籤的 rect 即為選項區 bbox

**Layer 2：圖片 bbox**
- `page.get_images()` 取得頁面上所有嵌入圖
- 選項圖通常排列整齊、尺寸相近
- 找出 4 張（或 3 張）相鄰、y 座標接近的圖片 → 連成大 bbox

**Layer 3：啟發式 + 邊距**
- 從 `(A)` 標籤位置往下、往左右擴張給定 padding
- 適用無 frame rect 的 fallback 場景

> [!IMPORTANT]
> bbox 必須覆蓋**整個選項區**，含 (A)(B)(C)(D) 標籤文字與其下方的圖，
> 不可裁掉任何選項。寧可多留白也不要少切。

### 3.3 三圖選項的 D 選項處理

| `options.D` 內容 | 處理方式 |
|----------------|---------|
| 有文字（parser 抓到） | D 顯示為文字，A/B/C 用裁切圖 |
| 空字串 | D 也在圖中，整個 A~D 一起裁切 |

決策邏輯：
1. 先查 JSON `options.D` 是否為空
2. 若 D 非空 → 裁切只需涵蓋 A-C 區域（D 用文字顯示）
3. 若 D 空 → 裁切涵蓋整個 A-D

實作時可加 `--dry-run` 參數先打印每題的裁切 bbox 與 D 狀態供人工檢核。

### 3.4 解析度與品質

- 渲染 DPI：200（與現有 `assets/{year}/pages/` 一致）
- 輸出 JPG quality：90
- 預期單張大小：寬 ~1200-1600px，高 ~400-700px

### 3.5 自動驗證

裁切完成的圖片，建議加入下列檢查項：

| # | 檢查 | 方法 |
|---|------|------|
| I1 | 檔案存在 | `os.path.exists` |
| I2 | 檔案非空 | `os.path.getsize > 5KB` |
| I3 | 解析度合理 | PIL `Image.open` 寬高在合理範圍 |
| I4 | 視覺檢查 | 人工開瀏覽器看一遍 8 題 |
| I5 | aspect 比 | 長寬比應在 1.5 ~ 5 之間（太寬/太窄代表切錯） |

## 4. Schema 修正

### 4.1 JSON 欄位

在題目 JSON 加入新欄位 `image_options`：

```jsonc
{
  "number": 29,
  "type": "單題",
  "stem": "...",
  "options": {
    "A": "",
    "B": "",
    "C": "",
    "D": ""
  },
  "image_options": "11529i.jpg",    // 新增：指向 web/pic/item/ 的檔名
  "image_options_full": true,        // 新增：true = 圖覆蓋 A-D 全部；
                                     //      false = 只有 A-C 是圖，D 是文字
  ...
}
```

- 只有圖片選項題才設 `image_options`，其他題為 `null`
- `image_options_full: true` → A/B/C/D 都用圖顯示
- `image_options_full: false` → A/B/C 用圖、D 顯示 `options.D` 文字

### 4.2 Parser 觸發條件

parser 在 `extract_question()` 結尾加一條檢查：

```python
def has_image_options(opts: dict) -> bool:
    """True if any option is empty (likely an image option)."""
    empties = [k for k,v in opts.items() if not v]
    return len(empties) >= 2  # 至少 2 個空選項才視為圖片題
```

只要偵測到「≥ 2 個空選項」即標記 `image_options = "{year}{q:02d}i.jpg"`。
裁切腳本之後只要掃描所有 `image_options` 非空的題目即可。

### 4.3 向後相容

- 對於其他非圖片題，`image_options` 一律為 `null`
- 前端先讀 `image_options`，若為 `null` 走原流程
- 現有單題 / 題組子題結構完全不變

## 5. 前端呈現

### 5.1 renderQuestion 分流

`app.js` 中 `renderQuestion()` 先檢查 `q.image_options`：

```javascript
renderQuestion(q) {
  if (q.image_options) return this.renderImageOptionQuestion(q);
  // ... 原有流程
}

renderImageOptionQuestion(q) {
  const src = 'pic/item/' + q.image_options;
  // 渲染題幹 + 一張大圖（含 A,B,C 標籤）
  // 若 q.image_options_full === false → 額外顯示 (D) q.options.D 文字
  // 點擊圖片可放大 lightbox
}
```

### 5.2 互動行為

| 功能 | 說明 |
|------|------|
| **單張選項圖** | 整個選項區一張 JPG，標籤 (A)(B)(C)(D) 都在圖內 |
| **作答方式** | 因為選項是圖，使用者無法單擊選項 → 改用 A/B/C/D 按鈕列 |
| **按鈕列** | 題目下方顯示 4 個圓形字母按鈕 `A B C D`，點擊作答 |
| **D 文字混合** | 若 `image_options_full === false`，(D) 顯示為文字選項而非按鈕 |
| **放大燈箱** | 點擊選項圖開 lightbox 看原圖（與現有圖表 lightbox 共用） |
| **題組子題** | 114 Q43 是題組 → 走題組三欄版型，選項區換成圖 + 按鈕列 |
