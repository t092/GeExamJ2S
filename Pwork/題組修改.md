# 題組子題 (Group Questions) 解析與排版改良建議

## 一、問題分析

### 1.1 題幹文字滲入選項的根本原因

在 PDF 文字流中，**題組的閱讀文（passage）出現在子題的「後方」**，而非前方。以 112 年第 52-54 題為例，原始文字流的順序為：

```
閱讀下列選文，回答第 52 至 54 題：
52. [題幹]     ← 子題先出現
    (A)... (B)... (C)... (D)...
53. [題幹]
    (A)... (B)... (C)... (D)...
54. [題幹]
    (A)... (B)... (C)...
    (D)以修法程序而言，皆可由行政機關自行修訂    ← 最後一個選項
美國的反體罰組織於1998年發起...                   ← 閱讀文緊接在後
```

> [!CAUTION]
> 目前 [parse_questions.py](file:///f:/OneDrive/grifon/OneDrive/AI/VibeVoding/JHexam/pipeline/parse_questions.py) 的 `extract_question()` 函式（第 93-132 行）在解析選項 D 時，使用的正則 `\(D\)\s*(.+?)(?=\([A-D]\)|\Z)` 會貪婪匹配到字串結尾 `\Z`，因此**把閱讀文整段吞進選項 D 的文字中**。

### 1.2 受影響範圍（所有年度）

| 年度 | 受影響題號 | 選項 D 錯誤內容 |
|------|-----------|---------------|
| 112 | Q48, Q51, Q54 | 每組的最後一題，D 選項末尾混入下一段閱讀文 |
| 113 | Q48, Q51, Q54 | 同上模式 |
| 114 | Q45, Q48, Q51, Q54 | 同上模式 |

> [!IMPORTANT]
> 每個題組的**最後一題**的**最後一個選項 (D)** 都會受影響，因為閱讀文在 PDF 文字流中總是出現在子題之後。

### 1.3 額外發現：題組缺少閱讀文（passage）

目前的 JSON schema 中，題組子題的 `stem` 只包含子題的問句，**完全沒有儲存共用的閱讀文**。這導致：
- 前端無法顯示題組的閱讀文
- 考生看不到題組的上下文，無法作答

## 二、改良建議

### 2.1 Parser 修正 — 分離選項與閱讀文

#### 方案：在 `parse_group_section()` 中先提取閱讀文，再解析子題

```
修改 parse_questions.py 的 parse_group_section() 函式
```

**核心策略：**

1. **先識別每個題組 block** 的邊界（`閱讀下列選文，回答第 X 至 Y 題`）
2. **在 block 內容中，找到最後一個 `(D)` 選項之後的文字** → 這就是閱讀文 passage
3. **將 passage 從 block 內容中分離**，避免它被吞進選項
4. **將 passage 儲存到 JSON** 的新欄位 `passage` 中

**具體作法：**

```python
def parse_group_section(text):
    # 1. 找出所有題組 block
    block_pattern = re.compile(
        r'閱讀下列選文，回答第\s*(\d+)\s*至\s*(\d+)\s*題[：:]?\s*\n(.*?)(?=閱讀下列選文|試題結束|\Z)',
        re.DOTALL
    )
    blocks = block_pattern.findall(text)

    questions = []
    for q_start, q_end, content in blocks:
        q_start, q_end = int(q_start), int(q_end)

        # 2. 把 content 拆分成：子題區域 vs 閱讀文
        #    找最後一題的 (D) 選項後的文字 → passage
        passage, clean_content = split_passage_from_questions(content, q_end)

        # 3. 解析子題（用清理後的 content）
        sub_qs = parse_section(clean_content, '題組子題')
        for q in sub_qs:
            q['group_range'] = [q_start, q_end]
            q['type'] = '題組子題'
            q['passage'] = passage  # 新增欄位
        questions += sub_qs

    return questions
```

**`split_passage_from_questions()` 的邏輯：**

```python
def split_passage_from_questions(content, last_q_num):
    """
    從題組 block 中，找到最後一題的最後一個選項 (D) 之後的文字，
    將其識別為閱讀文（passage），並從 content 中移除。
    """
    # 找最後一個 (D) 選項的位置
    d_positions = [m.start() for m in re.finditer(r'\(D\)', content)]
    if not d_positions:
        return '', content

    last_d_pos = d_positions[-1]
    # 從最後一個 (D) 開始，找到選項文字結束的位置
    # 選項文字結束的標誌：連續換行後，不再是選項格式的文字
    after_d = content[last_d_pos:]

    # 用啟發式規則：選項 D 的文字通常在同一行或下一行結束
    # 閱讀文通常是完整的段落，以句號結尾
    # 方法：找到 (D) 之後，下一個句號 + 換行的位置
    # 或者用更穩健的方式：找到不屬於選項格式的第一行

    # ... 詳見實作
    passage = content[passage_start:]
    clean_content = content[:passage_start]
    return passage.strip(), clean_content
```

> [!WARNING]
> 閱讀文與選項 D 的分界點不容易精確判斷，因為 PDF 文字流中沒有明確的分隔標記。需要結合多種啟發式規則（如：閱讀文通常以主語開頭、包含完整語句、長度較長等）。

### 2.2 JSON Schema 擴充

在現有 schema 中新增 `passage` 欄位：

```json
{
  "number": 52,
  "type": "題組子題",
  "group_range": [52, 54],
  "passage": "美國的反體罰組織於1998年發起「不打小孩日」的活動...",
  "stem": "關於文中活動理念對臺灣帶來影響的過程，最適合以下列何者說明？",
  "options": {"A": "...", "B": "...", "C": "...", "D": "..."},
  "figures": ["圖(十六)"]
}
```

> [!NOTE]
> 同一題組的所有子題（如 Q52、Q53、Q54）都共用相同的 `passage`，前端可以利用 `group_range` 來判斷何時需要顯示 passage。

### 2.3 前端排版 — 題組子題的三欄結構

目前前端只有兩種版型：
- **無圖題**：單欄（題幹 + 選項）
- **有圖題**：雙欄（左：題幹+選項 / 右：圖片）

題組子題需要一個專用的新版型：

#### 提議的三區塊結構

```
┌──────────────────────────────────────────────────────┐
│  ① 閱讀文 (passage)                                   │
│  [全寬，上方區域，背景色區分]                              │
│  美國的反體罰組織於1998年發起「不打小孩日」的活動...         │
└──────────────────────────────────────────────────────┘
┌─────────────────────────┬────────────────────────────┐
│  ② 圖片區              │  ③ 題目區                    │
│  [左欄，佔 40%]         │  [右欄，佔 60%]              │
│                         │                             │
│  圖(十六) 宣傳標語       │  52. 關於文中活動理念...       │
│  [可放大的圖片]          │  (A) 法律修改程序的變化        │
│                         │  (B) 科技發展帶來的衝擊        │
│  表(九) 登山路線資訊     │  (C) 風俗習慣的文化傳承        │
│  [可放大的圖片]          │  (D) 全球化下的文化交流        │
│                         │                             │
│                         │  ← 上一題 | 下一題 →          │
└─────────────────────────┴────────────────────────────┘
```

### 2.4 特殊情況處理

#### 無圖片的題組

部分題組沒有圖表（如 112 年 Q44-45），此時版型退化為：

```
┌──────────────────────────────────────────────────────┐
│  ① 閱讀文 (passage)                                   │
└──────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────┐
│  ② 題目區 (全寬)                                      │
│  44. 上文中「唐手佐久川」與王國的使節團得以前往中國...      │
│  (A)... (B)... (C)... (D)...                          │
└──────────────────────────────────────────────────────┘
```

#### 圖片跨子題共用

同一題組的多題可能引用不同圖表，左欄應顯示**該題組所有子題引用的所有圖表**，而非只顯示當前子題的圖表。

## 四、風險與注意事項

> [!WARNING]
> **PDF 文字流的順序不穩定**：PyMuPDF 的 `page.get_text()` 對多欄排版的頁面，文字流順序可能與視覺順序不同。需要實際測試每一題的 passage 分離結果。

> [!NOTE]
> **數字格式問題**：112 年的部分頁面（如 Page 13）有數字控制字元（`\u0016`, `\u0019` 等）混入文字流，可能影響 passage 的分離。需要在 parser 中加入清理邏輯。

> [!TIP]
> **建議先手動驗證 3 年 × 4 題組 = 12 組閱讀文**，確認 passage 分離的正確性後再進行前端改造。
