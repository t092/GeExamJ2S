# 社會科會考題庫 — 題組子題結構改良規劃

## 0. 摘要

國中教育會考社會科題本每年共 54 題，其中第 43–54 題為 **題組**（共 4 個 group，每個 group 3 題）。目前 `data/<year>.json` 對單題 (Q1–Q42) 解析無誤，但對題組子題 (Q43–Q54) 有兩類結構性錯誤：

1. **選文（passage）跑到末題選項 D**：以 112 年 Q52–54 為例，整段選文被併入 Q54 的 `options.D`。
2. **圖表歸屬錯位**：選文引用的圖（如 112 Q53 的 圖(十六)）被錯誤綁在子題 stem。

本文件提出**保留現 schema、只新增欄位**的改良方案，涵蓋 parser、JSON 結構與前端三欄式版型，並列明執行步驟、驗證清單與影響範圍。

---

## 1. 問題根因分析

### 1.1 PDF 文字流的題組排版

PyMuPDF 抽出的純文字其中 `二、題組` 區段的結構為：

```
閱讀下列選文，回答第 52 至 54 題：
52. <題目>
(A)<選項>
(B)<選項>
(C)<選項>
(D)<選項>
53. <題目>
(A)...
(B)...
(C)...
(D)...
54. <題目>
(A)...
(B)...
(C)...
(D)<末題選項 D 內容>
<整段選文，全文不分行編號>
圖(十六)
```

關鍵點：**選文出現在最後一題選項 (D) 之後**，且**整段沒有任何題號前綴**。這是 PDF 排版特性，PyMuPDF 忠實萃取後即如此。

### 1.2 現行 parser 的處理缺口

`pipeline/parse_questions.py`：

- `parse_group_section` (L67–91) 用 `block_pattern` 正確抓到整個 block（含選文），但第 85 行把整塊丟給 `parse_section`。
- `parse_section` (L49–65) 用 `r'(?:^|\n)(\d{1,2})\.(?:\s*\n|\s+)(.*?)(?=\n\d{1,2}\.\s|\Z)'` 切子題 body。**末題 (Q54) 後面沒有下一個題號**，于是 body 一路延伸到 `\Z`（block 結尾），**整段選文被吞入 Q54 的 body**。
- `extract_question` (L93–132) 的第 108 行 `re.findall(r'\(([A-D])\)\s*([^\(]+)', options_raw)` 取 (D) 的內容時，`[^\(]+` 不含 `(`，但選文內含 `「」`、`（）` 等全形括號，故**選文整段變成 (D) 的尾巴**。

### 1.3 受影響的題目

所有題組的**最後一題** (112 年：Q45 / Q48 / Q51 / Q54) 都受影響。差別只在於選文有沒有圖、末題 D 有沒有被裁切乾淨。檢視 `data/112.json` Q54 實際內容：

- `stem`：`"關於文末前、後所提及我國的二項法規，下列敘述何者最適當？"` ✅
- `options.D`：`"以修法程序而言，皆可由行政機關自行修訂美國的反體罰組織於1998年發起「不打小孩日」的活動..."` ❌（後半為整段選文）

此外，112 Q53 的 `figures: ["圖(十六)"]`，但 圖(十六) 實際屬於**選文**，應歸到 group 層級而非子題。

## 2. 改良方案總覽

### 2.1 設計原則

| 原則 | 內涵 |
|------|------|
| **schema 向後相容** | 保留現有欄位結構，前端舊邏輯仍可讀題作答 |
| **passage 一處保留** | 不在每個子題重複存選文，避免資料冗餘 |
| **圖表歸屬精準** | 選文圖表與子題圖表分開 |
| **parser 單點修正** | 不動單題流程，只在 `parse_group_section` 與 `extract_question` 增量修改 |
| **前端三欄式** | 題組獨立版型：上 passage、下左圖、下右題目，子題切換不重建 passage |

### 2.2 改良範圍

```
┌──────────────────────────────────────────────────┐
│ 1. Schema        │ data/<year>.json              │
│                  │  - 新增 group_id              │
│                  │  - 新增 passage (僅首題)       │
│                  │  - 新增 passage_figures        │
│                  │  - 修正 options.D 不含選文     │
├──────────────────┼───────────────────────────────┤
│ 2. Parser        │ pipeline/parse_questions.py   │
│                  │  - 切出 passage               │
│                  │  - 強化 (D) 截斷              │
│                  │  - 圖表重新歸屬                │
├──────────────────┼───────────────────────────────┤
│ 3. Frontend      │ web/css/style.css             │
│                  │ web/js/app.js                 │
│                  │  - 題組專用 CSS Grid 雙列       │
│                  │  - group_id 聯動子題切換       │
└──────────────────┴───────────────────────────────┘
```

## 3. Schema 改良細節

### 3.1 新欄位定義

保留現有：`number / type / group_range / stem / options / figures / figure_pages`

新增：

| 欄位 | 型別 | 存放位置 | 說明 |
|------|------|---------|------|
| `group_id` | string | 每個題組子題 | 形如 `"52-54"`，同 group 內子題一致 |
| `passage` | string \| null | 僅 group 第一子題 | 整段閱讀選文；其他子題為 `null` |
| `passage_figures` | string[] | 僅 group 第一子題 | passage 引用的圖表，如 `["圖(十六)"]` |
| `passage_figure_pages` | object | 僅 group 第一子題 | 圖表 → 頁碼映射 |

### 3.3 單題與舊欄位處理

- 單題 (Q1–Q42) **不變**，新增欄位均設為 `null` / `[]` / `{}`，前端讀舊欄位即可作答。
- `group_range` 仍保留，與 `group_id` 並存（`group_id = f"{group_range[0]}-{group_range[1]}"`），方便舊前端繼續依 `group_range` 分群。

## 4. Parser 改良細節

### 4.1 修改點

| 函式 | 位置 | 修改 |
|------|------|------|
| `parse_group_section` | L67–91 | 切出 passage 並寫入首題 |
| `extract_question` | L93–132 | 強化 (D) 選項截斷、不吞選文 |
| 題組圖表歸屬 | 新增輔助函式 | passage 圖 vs 子題圖分離 |

### 4.2 `parse_group_section` 演算法

```python
def parse_group_section(text):
    block_pattern = re.compile(
        r'閱讀下列選文，回答第\s*(\d+)\s*至\s*(\d+)\s*題[：:]?\s*\n(.*?)(?=閱讀下列選文|試題結束|\Z)',
        re.DOTALL
    )
    blocks = block_pattern.findall(text)
    questions = []

    for q_start, q_end, content in blocks:
        q_start, q_end = int(q_start), int(q_end)
        group_id = f"{q_start}-{q_end}"

        # 1. 用題號 split 子題 body（含末題延伸到選文）
        sub_qs = parse_section(content, '題組子題')
        sub_qs.sort(key=lambda x: x['number'])

        # 2. 取出選文：末題 body 截到 (D) 結束，剩餘即 passage
        #    重新對末題做 cut，避免原本 body 吞入選文
        last_q_num = sub_qs[-1]['number']
        last_q_start_idx = find_q_start_index(content, last_q_num)
        last_q_body_raw = content[last_q_start_idx:]

        # 找 (D) 選項後第一個「非選項」位置作為 passage 起點
        passage_start = find_passage_start(last_q_body_raw)  # 見 4.3
        passage_tail = last_q_body_raw[passage_start:]

        # 開頭若有選文（罕見，少數年份 passage 在題目之前）
        first_q_start_idx = find_q_start_index(content, sub_qs[0]['number'])
        passage_head = content[:first_q_start_idx]
        # 去掉閱讀下列選文…etc 引導句（已被 block_pattern 消耗，通常為空）

        passage = (passage_head + passage_tail).strip()

        # 3. 修正末題 options.D（已由 extract_question 處理截斷）
        # 4. 圖表歸屬
        passage_figs = list(set(re.findall(r'[圖表]\([一二三四五六七八九十\d]+\)', passage)))
        for q in sub_qs:
            q['group_range'] = [q_start, q_end]
            q['type'] = '題組子題'
            q['group_id'] = group_id
            # 子題 figures 移除 passage 已认領的
            q['figures'] = [f for f in q['figures'] if f not in passage_figs]

        # 5. passage 寫入首題
        first = sub_qs[0]
        first['passage'] = passage
        first['passage_figures'] = passage_figs
        first['passage_figure_pages'] = resolve_pages(passage_figs)  # 從 figures.json 查
        for other in sub_qs[1:]:
            other['passage'] = None
            other['passage_figures'] = []
            other['passage_figure_pages'] = {}

        questions += sub_qs

    return questions
```

### 4.3 `extract_question` 強化選項截斷

現行 (L108)：
```python
matches = re.findall(r'\(([A-D])\)\s*([^\(]+)', options_raw)
```

問題：`[^\(]+` 不含半形 `(`，但中文全形括號 `（）`、`「」` 都會被吞。

修正為**以結構邊界切斷**：
```python
matches = re.findall(
    r'\(([A-D])\)\s*(.*?)(?=\n\([A-D]\)|\n\d{1,2}\.\s|\Z)',
    options_raw,
    re.DOTALL
)
```

含義：
- `(A)` 內容取到「下一個 `\n(B)`」或「下一題題號」或「字串結尾」為止
- 末題 (D) 因後面接著是選文（沒有 `\n(B)` 也沒有下一題號）→ 第 108 行的 `(D)` 仍會一路吃到 `\Z`，**這正是要避免的**

因此**必須配合 `parse_group_section` 的**「先從 content 切出 passage_tail，再交給 `extract_question` 限縮 options_raw 範圍」**，雙重保險：

```python
# 在 parse_group_section 末題處理中：
last_q_body_for_parse = last_q_body_raw[:passage_start]    # 只到選項結束
last_q = extract_question(last_q_num, last_q_body_for_parse, '題組子題')
# 不再用 parse_section 切末題，避免吞選文
```

### 4.4 圖表歸屬規則

| 圖表 label 出現位置 | 歸屬 |
|--------------------|------|
| 同時在 passage 與子題 stem | passage_figures（優先） |
| 僅在 passage | passage_figures |
| 僅在子題 stem | 該子題 figures |
| 同 group 多子題引用同圖 | passage_figures |

`figures` 與 `figure_pages` 仍依現有 `data/<year>_figures.json` 查頁碼，不另行修改 figures 對應檔。

### 4.5 風險與保護

- **備份**：每次重跑前將 `data/<year>.json` 複製為 `data/<year>.json.bak`，避免覆寫失誤。
- **可逆驗證**：新增 `pipeline/check_groups.py`（建議）做下列檢查。若未新增腳本，至少在 `check_quality.py` 增列檢查項。
- **單題保護**：`parse_section` 不變，單題解析路徑完全不受影響。

## 5. 前端改良細節

### 5.1 CSS Grid 雙列版型

```css
/* 題組版型 */
.group-question {
  display: grid;
  grid-template-areas:
    "passage passage"
    "fig     stem";
  grid-template-columns: minmax(140px, 1fr) 2fr;
  gap: 16px;
}
.group-question .passage { grid-area: passage; }    /* 跨雙欄，上方整列 */
.group-question .fig     { grid-area: fig;     }    /* 左下圖 */
.group-question .stem    { grid-area: stem;    }    /* 右下題目 */

/* 手機直排 */
@media (max-width: 720px) {
  .group-question {
    grid-template-areas: "passage" "fig" "stem";
    grid-template-columns: 1fr;
  }
}
```

### 5.2 題組 DOM 結構（示意）

```html
<div class="group-question" data-group-id="52-54">
  <div class="passage">
    <h4>閱讀選文</h4>
    <p>{{ passage }}</p>
  </div>
  <div class="fig">

      <button @click="nextInGroup">下一題</button>
    </div>
  </div>

```

### 5.3 app.js 切換邏輯

載入某一題組子題時：

1. 依 `q.group_id` 找同 group 所有子題
2. 從 group **首題** 取 `passage`、`passage_figures`、`passage_figure_pages`
3. 渲染上方 `passage` 區塊一次
4. 渲染左下 `fig` 區：先放 `passage_figures`；若當前子題也有 `figures`，疊加顯示（或切換時換圖）
5. 渲染右下 `stem` 區：題號、題目、選項
6. 同 group 切題（52→53→54）時**不重建 passage 區**，只更新右下題目；左下圖若 passage 圖與子題圖重疊則保留，否則切圖

單題（type === '單題'）沿用現有 layout，不套用 `.group-question` 容器。

### 5.4 子題導覽

- 題組子題顯示「第 X 題 / 共 N 題」提示與切換按鈕
- 切換時若使用者已作答，記錄在 localStorage / 記憶體狀態
- 整份測驗的「下一題」走到題組邊界時自動跳到同 group 下一子題，跨越 group 才前進到新題組

### 已知風險

1. **passage 切點誤判**：選文末尾若緊接下一題題號格式特殊（罕見），可能切不乾淨。緩解：用「(D) 選項結束後第一個**非選項格式**且**非題號**的連續段落」作為 passage 起點；前後以句號、換行驗證。
2. **跨頁選文**：PyMuPDF 抽字時跨頁可能插入 `=== PAGE N ===` 與空白行。`parse_questions.py` 已先 `re.sub(r'=== PAGE \d+ ===', '', raw)` 做掉，但要確認題組選文跨頁沒有殘留樣板字（如「請翻頁繼續作答」）。
3. **罕見題組 passage 在題目之前**：如某些年份選文排在前、題目在後。`parse_section` 會把開頭到第一題題號前的內容視為前一題的 body 尾巴（但題組沒有前一題，因此會丟失）。`parse_group_section` 改寫時要顯式抓 `passage_head = content[:first_q_start_idx]` 來保留。
4. **圖表 label 同名跨 group**：理論上不會發生（每 group 內圖表編號唯一），但若有，需以 group_id 隔離查 `figure_pages`。
