### 1.3 核心問題

從 PDF 實際頁面分析，社會科圖表分為 **5 種類型**，其中只有 2 種能被 `get_images()` 抽出：

| 圖表類型 | 範例 | `get_images()` 可抽取？ | 整頁渲染可見？ |
|---------|------|----------------------|-------------|
| **地圖（點陣圖）** | 圖(一)「挖仔聚落地圖」 | ✅ 可抽出 | ✅ |
| **歷史照片/版畫** | 圖(三)「木刻版畫」 | ✅ 可抽出 | ✅ |
| **表格（文字排版）** | 表(一)「女性六項權利」、表(二)「電視台報導比例」 | ❌ 純文字，非嵌入圖 | ✅ |
| **示意圖（向量繪製）** | 圖(四)「親屬關係圖」 | ❌ 向量路徑，非嵌入圖 | ✅ |
| **統計圖（向量繪製）** | 長條圖、折線圖等 | ❌ 向量路徑，非嵌入圖 | ✅ |

> [!IMPORTANT]
> `page.get_images()` 只能抽出**嵌入的點陣圖**（地圖、照片），但會考社會科大量圖表是**向量繪製或純文字排版的表格**，無法被抽出。唯一 100% 可靠的方案是**整頁渲染 + 裁切**。

### 2.1 Layer 1：整頁圖片裁切（主方案）

**原理**：找到圖表標籤（如 `圖(一)`）在 PDF 頁面上的座標位置，向上搜尋圖片/空白區域的上界，裁切出完整圖表區域。

**實作步驟**：

```python
import fitz

def crop_figure(pdf_path, fig_id, fig_page, dpi=200):
    """從整頁中裁切出指定圖表區域。"""
    doc = fitz.open(pdf_path)
    page = doc[fig_page - 1]  # 0-indexed
    zoom = dpi / 72

    # Step 1: 找到圖表標籤的位置
    blocks = page.get_text("dict")["blocks"]
    label_bbox = None
    for b in blocks:
        if b["type"] != 0:
            continue
        for line in b.get("lines", []):
            text = "".join(s["text"] for s in line.get("spans", []))
            if text.strip() == fig_id:
                label_bbox = line["bbox"]
                break

    if not label_bbox:
        return None

    # Step 2: 確定裁切範圍
    label_y = label_bbox[1]      # 標籤的上邊界 y
    label_x_center = (label_bbox[0] + label_bbox[2]) / 2
    label_bottom = label_bbox[3]  # 標籤的下邊界 y

    # 向上搜尋：找到圖表的上邊界
    # 策略：從標籤往上掃，找到最近的「題目文字行」作為上界
    text_lines_above = []
    for b in blocks:
        if b["type"] != 0:
            continue
        for line in b.get("lines", []):
            line_y = line["bbox"][3]  # 行的下邊界
            if line_y < label_y:       # 在標籤之上
                text = "".join(s["text"] for s in line.get("spans", []))
                # 排除圖表內部的文字（通常字號較小或位置偏右）
                text_lines_above.append({
                    "y": line_y,
                    "text": text.strip(),
                    "bbox": line["bbox"]
                })

    # 找圖表區域的上界：最近一個「題目選項行」的底部
    # 或者使用嵌入圖片的上邊界
    images_on_page = page.get_image_info()
    img_tops = [img["bbox"][1] for img in images_on_page
                if abs((img["bbox"][0] + img["bbox"][2])/2 - label_x_center) < 150
                and img["bbox"][1] < label_y]

    if img_tops:
        top_bound = min(img_tops) - 5  # 圖片最上方 - 5pt 邊距
    else:
        top_bound = label_y - 200      # fallback: 標籤往上 200pt

    # Step 3: 設定裁切寬度（圖表通常在頁面右半或全寬）
    page_width = page.rect.width
    margin = 30
    x0 = max(label_bbox[0] - 50, margin)
    x1 = min(page_width - margin, label_bbox[2] + 200)

    # Step 4: 裁切
    crop_rect = fitz.Rect(x0, top_bound, x1, label_bottom + 5)
    mat = fitz.Matrix(zoom, zoom)
    pix = page.get_pixmap(clip=crop_rect, matrix=mat)

    return pix  # 呼叫方 pix.save("output.webp")
```

**優缺點**：
- ✅ 適用所有圖表類型，包括向量圖和文字表格
- ✅ 不依賴嵌入圖片是否存在
- ⚠️ 裁切邊界可能不精確，需要人工微調或 LLM 輔助確認
- ⚠️ 圖表與題目文字交錯時，裁切區域可能包含多餘文字

### 2.3 Layer 3：文字表格結構化重建

**原理**：用 pdfplumber 的 `page.extract_tables()` 抽取表格數據，存入 JSON，前端用 HTML `<table>` 渲染。

**實作**：

```python
import pdfplumber

def extract_table(pdf_path, page_num):
    """從指定頁面抽取表格數據。"""
    with pdfplumber.open(pdf_path) as pdf:
        page = pdf.pages[page_num - 1]  # 0-indexed
        tables = page.extract_tables()
        results = []
        for table in tables:
            if table and len(table) > 1:
                results.append({
                    "headers": table[0],
                    "rows": table[1:]
                })
        return results
```

**JSON 儲存格式**：

```json
{
  "number": 15,
  "stem": "表(二)是恬恬為了製作課堂報告所蒐集的資料...",
  "figures": ["表(二)"],
  "figure_assets": {
    "表(二)": {
      "type": "表格",
      "source": "structured",
      "table_data": {
        "caption": "占播報時間比例 (%)",
        "headers": ["電視臺", "總比例", "正面報導", "負面報導"],
        "rows": [
          ["水果臺", "36", "28", "4"],
          ["F News", "12", "4", "6"],
          ["西水新聞", "7", "3", "2"],
          ["W 電視", "28", "4", "21"]
        ],
        "footnote": "統計時間：5/1-5/7每日19:00~20:00的新聞"
      },
      "fallback_page": "assets/114/pages/page_04.png",
      "page_number": 4
    }
  }
}
```

**前端渲染**：

```javascript
function renderTable(tableData) {
  let html = `<table class="exam-table">`;
  if (tableData.caption) {
    html += `<caption>${tableData.caption}</caption>`;
  }
  html += `<thead><tr>`;
  for (const h of tableData.headers) {
    html += `<th>${h}</th>`;
  }
  html += `</tr></thead><tbody>`;
  for (const row of tableData.rows) {
    html += `<tr>`;
    for (const cell of row) {
      html += `<td>${cell}</td>`;
    }
    html += `</tr>`;
  }
  html += `</tbody></table>`;
  if (tableData.footnote) {
    html += `<p class="table-footnote">${tableData.footnote}</p>`;
  }
  return html;
}
```

**優缺點**：
- ✅ 可搜尋、可縮放、響應式、無障礙
- ✅ 小螢幕（手機）體驗遠優於圖片
- ⚠️ pdfplumber 表格抽取對合併儲存格的支援有限
- ⚠️ 複雜表格可能需要人工校正

## 3. JSON schema 改進

### 3.1 新增 `figure_assets` 欄位

在現有 `figures` 陣列（文字引用）之外，新增 `figure_assets` 物件，提供圖表的實際資產路徑：

```json
{
  "number": 4,
  "type": "單題",
  "group_range": null,
  "stem": "圖(一)是某行政區內名為挖仔的聚落地圖。根據該聚落所處環境特色判斷，其地名代表的意義最可能為下列何者？",
  "options": {
    "A": "位於岬灣海岸的灣澳",
    "B": "鄰近溪流河道的轉彎處",
    "C": "位處高山之間的小谷地",
    "D": "台地上的低窪地蓄水成湖"
  },
  "figures": ["圖(一)"],

  "figure_assets": {
    "圖(一)": {
      "type": "地圖",
      "source": "embedded",
      "image_path": "assets/114/figures/fig01.webp",
      "fallback_page": "assets/114/pages/page_02.png",
      "page_number": 2
    }
  }
}
```

### 3.2 `figure_assets` 欄位定義

| 欄位 | 類型 | 說明 |
|------|------|------|
| `type` | string | 圖表類型：`"地圖"` `"照片"` `"統計圖"` `"示意圖"` `"表格"` `"版畫"` `"漫畫"` `"史料"` |
| `source` | string | 圖片來源：`"embedded"`（嵌入圖直接用）、`"crop"`（整頁裁切）、`"structured"`（結構化表格） |
| `image_path` | string \| null | 裁切/嵌入圖的路徑，`source=structured` 時為 null |
| `fallback_page` | string | 整頁渲染圖路徑（保底顯示方案） |
| `page_number` | int | PDF 頁碼 |
| `table_data` | object \| null | 僅 `source=structured` 時有值，含 `headers`、`rows`、`caption`、`footnote` |

### 3.3 題組共用圖表的處理

題組（如第 52–54 題共用圖(二十一)）中，`figure_assets` 只需要在**題組第一題**定義一次，其餘子題引用同一個 `figures` 陣列：

```json
// 題 52（題組首題）— 定義完整的 figure_assets
{
  "number": 52,
  "type": "題組子題",
  "group_range": [52, 54],
  "stem": "...",
  "figures": ["圖(二十一)"],
  "figure_assets": {
    "圖(二十一)": {
      "type": "地圖",
      "source": "crop",
      "image_path": "assets/114/figures/fig21.webp",
      "fallback_page": "assets/114/pages/page_15.png",
      "page_number": 15
    }
  }
}

// 題 53（題組子題）— 引用但不重複定義
{
  "number": 53,
  "type": "題組子題",
  "group_range": [52, 54],
  "stem": "依據上文及圖(二十一)中的資訊判斷...",
  "figures": ["圖(二十一)"],
  "figure_assets": {}
}
```

前端邏輯：渲染題 53 時，若 `figure_assets` 為空但 `figures` 非空，往前查找同 `group_range` 的首題取得資產。

## 4. 前端顯示邏輯

### 4.1 優先順序

```javascript
function renderFigure(figId, figAsset) {
  // 1. 結構化表格 → 用 HTML table 渲染
  if (figAsset.source === 'structured' && figAsset.table_data) {
    return renderTable(figAsset.table_data);
  }

  // 2. 有裁切圖/嵌入圖 → 顯示圖片，失敗時 fallback 整頁
  if (figAsset.image_path) {
    const img = document.createElement('img');
    img.src = figAsset.image_path;
    img.alt = figId;
    img.onerror = () => {
      img.src = figAsset.fallback_page;
      img.classList.add('full-page-fallback');
    };
    return img;
  }

  // 3. 僅有整頁圖 → 直接顯示
  if (figAsset.fallback_page) {
    const img = document.createElement('img');
    img.src = figAsset.fallback_page;
    img.alt = figId;
    img.classList.add('full-page-fallback');
    return img;
  }
}
```
