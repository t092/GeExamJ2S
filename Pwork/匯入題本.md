## Step 2：Extract Text → `pipeline/output/{year}_text.txt`

```bash
python pipeline/extract_all.py
```

**原理：** PyMuPDF `page.get_text()` 逐頁讀取，輸出 `=== PAGE N ===` 分隔的純文字。

**115 年結果：** 15 pages → `pipeline/output/115_text.txt`（721 行）

**關鍵技術事實：** 必須使用 PyMuPDF（fitz），不可用 pdfplumber（CID 編碼 CJK 字型會變亂碼）。
*   **排版方向性提示**：PyMuPDF `page.get_text()` 預設以區塊（blocks）順序讀取文字。如果未來題本有出現多欄排版（Double Columns）或插圖交錯的情形，需特別注意文字串流是否交錯，必要時應在腳本中改為按座標排序（例如 `sort=True`）。

## Step 3：Parse Questions → `data/{year}.json`

```bash
python pipeline/parse_questions.py pipeline/output/{year}_text.txt
```

**原理：** `parse_questions.py` 執行以下步驟：

1. **去除頁標記與說明頁**：移除 `=== PAGE N ===`、`請翻頁繼續作答`、`試題結束`；從 `一、單題` 開始截取。
2. **分割單題/題組**：以 `二、題組` 為界切分。
3. **單題解析**（`parse_section`）：
   - Regex `r'(?:^|\n)(\d{1,2})\.(?:\s*\n|\s+)(.*?)(?=\n\d{1,2}\.\s|\Z)'`（DOTALL）
   - Q1-Q9：`\nN.\n`（題號獨佔一行）
   - Q10+：`\nN. `（題號 inline）
   - 從 `(A)` 標記切分 stem 與選項
4. **題組解析**（`parse_group_section`）：
   - Regex block：`閱讀下列選文，回答第 X 至 Y 題`
   - `split_passage_and_questions()` 分離選文與子題
   - 選文在最後一個 `(D)` 之後（常見），或第一個題號之前（罕見，如 113 Q44-45）
   - `find_passage_boundary()` 跳過表格數據、頁碼、控制字元後找到真實選文起點
   - 選文中的圖表歸入 `passage_figures`/`passage_tables`，與子題的 `figures`/`tables` 分離
5. **選項尾端清理**（`clean_option_tail`）：去除誤入選項文字的 `圖`/`表` 標籤片段、頁碼數字
6. **控制字元清理**：`re.sub(r'[\x00-\x09\x0b-\x1f]', '', text)`（保留 `\n`）

**115 年結果：** 54/54 questions，0 missing。

**115 年注意事項：** 本年單題範圍為 (1～43題)，比以往 (1～42) 多一題。Parser 不依賴固定範圍，會正確識別。

> **Skill 化建議：** parser 目前硬編 `load_figure_page_map(year)` 在 parse 階段取得 figure_pages。
> 但 `extract_question` 回傳時 `figure_pages` 固定為 `{}`，真正填入依賴後續 `merge_figures.py`。
> 建議在 Skill 中整合為一步，或在文件明確標示此依賴關係。

## Step 4：Extract Images → `assets/{year}/`

```bash
python pipeline/extract_images.py
```

**原理：** PyMuPDF 逐頁：
- `page.get_pixmap(matrix=fitz.Matrix(zoom, zoom))` → 200 DPI 全頁 PNG 存入 `assets/{year}/pages/page_NN.png`
- `page.get_images(full=True)` + `doc.extract_image(xref)` → 嵌入圖片（>50x50pt）存入 `assets/{year}/figures/pNN_imgMM.{ext}`

**115 年結果：** 15 page renders + 10 embedded figures。

**必要條件：** crop 步驟需要 page renders 已存在。extract_images 必須在 crop 之前執行。

## Step 5：Map Figures → `data/{year}_figures.json`

```bash
# 目前 map_figures.py 硬編年份 [112,113,114]，115 年需手動執行
# 以下為等效邏輯
python pipeline/map_figures.py  # 只處理 112-114，需修改或手動對 115 執行
```

**原理：** `find_figure_pages(pdf_path)` 逐頁搜尋：
- `page.get_text("dict")["blocks"]` 中獨立的標籤文字：`圖(N)`、`表(N)`
- 全文 inline regex：`r'[圖表]\([一二三四五六七八九十\d]+\)'`
- 每個 figure_id 記錄出現的頁碼集合，取最早出現頁為 `primary_page`

**輸出格式：**
```json
[
  {"id": "圖(一)", "pages": [2], "primary_page": 2, "type": "圖"},
  {"id": "表(一)", "pages": [3], "primary_page": 3, "type": "表"}
]
```

**115 年遇到的問題：** `map_figures.py` 第72行 `for year in ['112', '113', '114']:`，不包含 115。

**解決方式：** 手動執行等效 Python 邏輯（寫入 `data/115_figures.json`）：
```python
import json, fitz, re, os
from collections import defaultdict
ROOT = r'C:\path\to\JHexam'
DATA = os.path.join(ROOT, 'data')
pdf = os.path.join(ROOT, '115年國中教育會考社會科題本.pdf')
doc = fitz.open(pdf)
fig_pages = defaultdict(set)
for n in range(len(doc)):
    blocks = doc[n].get_text('dict')['blocks']
    for b in blocks:
        if b['type'] == 0:
            for line in b.get('lines', []):
                text = ''.join(s['text'] for s in line.get('spans', [])).strip()
                m = re.match(r'^[圖表]\([一二三四五六七八九十\d]+\)$', text)
                if m: fig_pages[m.group(0)].add(n + 1)
    inline = re.findall(r'[圖表]\([一二三四五六七八九十\d]+\)', doc[n].get_text())
    for fid in inline: fig_pages[fid].add(n + 1)
doc.close()
result = [{"id": fid, "pages": sorted(ps), "primary_page": sorted(ps)[0],
           "type": "圖" if fid.startswith('圖') else "表"}
          for fid, ps in sorted(fig_pages.items())]
with open(os.path.join(DATA, '115_figures.json'), 'w', encoding='utf-8') as f:
    json.dump(result, f, ensure_ascii=False, indent=2)
```

> **Skill 化建議：** `map_figures.py` 應改為接受年份參數或自動掃描所有 `data/{year}.json` 對應的 PDF。

## Step 6：Merge Figure Pages into Question JSON

```bash
# 目前 merge_figures.py 也硬編年份 [112,113,114]
# 對新年份需手動執行等效邏輯
```

**原理：** 將 `data/{year}_figures.json` 中的 `primary_page` 合併回題庫 JSON：
- `question["figure_pages"]`：對 `q["figures"]` 中的每個 figure_id 填入頁碼
- `question["table_pages"]`：對 `q["tables"]` 中的每個 table_id 填入頁碼
- `question["passage_figure_pages"]`：對 `q["passage_figures"]` 中的每個 figure_id 填入頁碼
- `question["passage_table_pages"]`：對 `q["passage_tables"]` 中的每個 table_id 填入頁碼

**手動等效邏輯（115 年）：**
```python
import json, os
DATA = r'C:\path\to\JHexam\data'
with open(os.path.join(DATA, '115.json'), encoding='utf-8') as f:
    qs = json.load(f)
with open(os.path.join(DATA, '115_figures.json'), encoding='utf-8') as f:
    fig_map = {fi['id']: fi for fi in json.load(f)}

for q in qs:
    q['figure_pages'] = {f: fig_map[f]['primary_page']
                         for f in q.get('figures', []) if f in fig_map}
    q['table_pages'] = {t: fig_map[t]['primary_page']
                        for t in q.get('tables', []) if t in fig_map}
    if q.get('passage_figures'):
        q['passage_figure_pages'] = {f: fig_map[f]['primary_page']
                                     for f in q['passage_figures'] if f in fig_map}
    if q.get('passage_tables'):
        q['passage_table_pages'] = {t: fig_map[t]['primary_page']
                                    for t in q['passage_tables'] if t in fig_map}

with open(os.path.join(DATA, '115.json'), 'w', encoding='utf-8') as f:
    json.dump(qs, f, ensure_ascii=False, indent=2)
```

> **Skill 化建議：** `merge_figures.py` 也應改為接受年份參數。
> 建議將 Step 4+5+6（extract_images → map_figures → merge）整合為一個
> `process_images.py <year>` 統一入口。

## Step 7：Validate Parse Quality → `check_groups.py`

```bash
python pipeline/check_groups.py 115
```

**檢查項目（10項）：**

| 檢查 | 內容 | 115 結果 |
|------|------|----------|
| C1_count | 總題數 = 54 | PASS |
| C2_D_clean | 每題組最後一題的 D 選項 ≤ 40 字元（無選文洩漏） | PASS |
| C3_gid_consistent | 同題組內所有子題的 group_id 一致 | PASS |
| C4_passage_present | 每題組子題皆有 >30 字的 passage | PASS |
| C5_passage_same | 同題組內 passage 完全相同 | PASS |
| C6_passage_figs_in_text | passage_figures 中的圖表確實在 passage 文字中出現 | PASS |
| C7_figs_no_overlap | 子題 figures 不與 passage_figures 重疊 | PASS |
| C8_single_stable | 單題內容與 .bak 一致（需有備份檔） | SKIP |
| C9_no_control_chars | 無殘留控制字元（`\x00-\x1f`） | PASS |
| C10_option_tail_clean | 選項文字尾端無孤立 `圖`/`表` 標籤 | PASS |
| C11_empty_options | 114 Q43 特殊圖像選項檢查（僅114） | SKIP |

若 C4_passage_present FAIL：表示 `split_passage_and_questions` 未能找到選文。需檢查：
- 題組區塊中 `閱讀下列選文…` 是否被正確匹配
- `find_passage_boundary` 的 CJK 字數閾值（≥10）對該選文是否過高
- 是否有罕見的選文-before-questions 佈局未被策略 B 捕捉

> **結構驗證提示**：目前 `C1_count` 檢驗總題數必須剛好為 54。若未來會考制度調整總題數，此檢驗應彈性讀取題本開頭之描述（如 `一、單題：(1～N題)`）來作為驗證總數的基準。

## Step 8：Crop Figures & Tables → `web/pic/{year}*.jpg`

```bash
python ~/.config/opencode/ext-skills/jhexam-figure-crop/scripts/crop_figures.py <project_root> <year>
```

**原理（詳見 jhexam-figure-crop SKILL.md）：**

**Figure (圖) chain（3+1）：** frame rect → embedded images → drawing clusters → heuristic crop

**Table (表) chain（5+post）：** lines (Method 0) → frame rect → embedded → cluster → heuristic → verify_and_extend

**115 年結果：** 16 figures + 9 tables，全部 9 個表均被 Method 0 (lines-based) 成功偵測。

**輸出：**
- `web/pic/115p01.jpg` ~ `115p16.jpg`（16 figures）
- `web/pic/115t01.jpg` ~ `115t09.jpg`（9 tables）
- `web/pic/figures_115.json`（crop 元資料：bbox、method、size）

**必要條件：** Step 4（extract_images）必須先完成，因為 crop 需要 `assets/{year}/pages/page_NN.png`。

### 裁切異常與降級覆寫處理
由於圖表裁切演算法（尤其是複雜表格與線條混合排版）在極端情況下可能產生裁切不全或出錯：
1.  **手動覆寫 (Manual Overwrite)**：可直接透過手動截圖，將正確的圖表檔案覆蓋至 `web/pic/{year}p{NN}.jpg` 或 `{year}t{NN}.jpg`。
2.  **降級全頁顯示 (Fallback to Full Page)**：若裁切效果差且無暇手動截圖，可直接在資料 JSON 中刪除對應圖表的頁碼映射（如 `figure_pages` 或 `table_pages` 內的鍵值對）。前端在找不到切圖後，會自動降級讀取原始的整頁 PNG 渲染圖並提供右鍵查看原始頁面提示。

## 本次處理中遇到的問題與解法

### 問題 1：PDF 檔名不符合 glob 模式
- **現象：** 原始檔 `115P_Society.pdf`，`extract_all.py` 的 glob `*年國中教育會考社會科題本.pdf` 無法匹配
- **解法：** `Rename-Item` 改為 `115年國中教育會考社會科題本.pdf`
- **Skill 化建議：** 接受 `--pdf <path>` 參數，不必強制命名規則

### 問題 2：`map_figures.py` 只處理 112-114
- **現象：** 第72行 `for year in ['112', '113', '114']:` 硬編
- **解法：** 手動執行等效 Python 邏輯產出 `data/115_figures.json`
- **Skill 化建議：** 加入年份參數，或自動偵測 `data/` 下的 `{year}.json`

### 問題 3：`merge_figures.py` 也只處理 112-114
- **現象：** 第10行 `for year in ['112', '113', '114']:` 硬編
- **解法：** 手動執行等效 Python 邏輯，將 figure_pages/table_pages 合入 `115.json`
- **Skill 化建議：** 整合 Step 4+5+6 為統一入口

### 問題 4：`figure_pages` 填入流程分散在三個 script
- **現象：** `parse_questions.py`（Step 3）產生 `figure_pages: {}`，`map_figures.py`（Step 5）產出 mapping，`merge_figures.py`（Step 6）才真正填入。新手易遺漏 Step 5-6。
- **解法：** 在流程文件中明確標示此依賴鏈
- **Skill 化建議：** 將 Step 3+5+6 整併入 `parse_questions.py`，parse 時一併讀取 figure_map 並寫入 pages（`load_figure_page_map` 已有此邏輯但未用於 `figure_pages`，僅用於 `passage_figure_pages`）

### 問題 5：PowerShell inline Python 的中文字串跳脫
- **現象：** `python -c "..."` 中包含中文字元時容易觸發語法錯誤（引號被解析器誤判）
- **解法：** 改用獨立 `.py` 檔案執行
- **Skill 化建議：** 避免在 skill 中使用 inline Python；所有邏輯放入獨立 script

### 問題 6：115 年單題範圍變化 (1～43 vs 以往 1～42)
- **現象：** 題本寫 `一、單題：(1～43題)` 而非以往的 (1～42題)
- **解法：** parser 僅依賴 `二、題組` 分割，不受影響。注意到總題數仍為 54（43 單題 + 11 題組，取代原有的 42+12）。
- **Skill 化建議：** parser 不應假設固定題數範圍
