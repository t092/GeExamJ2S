# 選項為圖形時的處理方法

> 適用場景：國中教育會考社會科題本中，部分題目的選項以點陣圖（地圖、符號、印章等）
> 呈現，PyMuPDF 文字萃取層無法取得選項內容。本文記錄從 parser 標記、PDF 定位裁切、
> 到前端渲染的完整解決方案。

---

## 0. 問題背景

國中教育會考社會科 PDF 題本中，某些題目的 A/B/C/D 選項不是文字，而是圖片——
例如手繪地圖、郵票圖案、符號圖示。PyMuPDF `page.get_text()` 只能萃取文字向量，
圖片選項在文字流中空白，parser 無法取得選項文字，JSON 中 `options.A/B/C/D` 為空
字串（或 fallback regex 誤抓為 `(B)`、`(C)`、`(D)` 等偽文字），導致前端無法
顯示選項。

**解法**：將該題的整個選項區（含 (A)(B)(C)(D) 標籤與下方圖形）一次裁切為單張
JPG，前端改以圖片 + 字母按鈕取代文字選項列表。

### 1.1 選項版面類型

| 類型 | 排版 | 範例 |
|------|------|------|
| **A** | 四圖並排，下方各帶標籤 `(A)(B)(C)(D)` | 115 Q29 |
| **B** | 三圖並排 + 一個文字選項（D 是文字） | 112 Q39、114 Q29 |

目前實測 7 題的 `image_options_full` 全為 `true`（A~D 全是圖），類型 B
尚未實際出現，但程式碼已預留支援（見 §3.3）。

### 1.2 為什麼 parser 的文字萃取失敗

1. **點陣圖無文字向量**：PyMuPDF 只能萃取 glyph 文字，圖片中的地圖、符號
   不在文字流中。
2. **標籤被跳過**：`(A)` `(B)` 雖是文字，但夾在圖片中央，位置難以與文字流
   排序；PyMuPDF 常因其 bbox 與圖片重疊而以「圖片內文字」跳過。
3. **Fallback regex 誤抓**：parser 的 fallback regex `\(([A-D])\)\s*(.+?)`
   在找不到選項內容時，會把「下一選項的標籤」當成內容抓進來——
   例如 A 的內容變成 `(B)`、B 變成 `(C)`、C 變成 `(D)`。

## 4. Phase 1：Parser 端 — 偵測與標記

### 4.1 核心函式 `has_image_options()`

檔案：`pipeline/parse_questions.py:58-88`

```python
def has_image_options(opts: dict) -> bool:
    """True if options are image-based (not extractable as text).

    Two signals:
      1. ≥ 2 options are literally empty
      2. All non-empty options look like fallback-grabbed junk — short strings
         that are just the next option label, e.g. '(B)', '(C)', '(D)'.
         When the real option content is an image, the fallback regex only
         captures the next label as text.
    """
    vals = [str(v).strip() for v in opts.values()]
    empty = [k for k, v in opts.items() if not v or not str(v).strip()]
    if len(empty) >= 2:
        return True
    # Fallback-grab detector: pattern where A=(B), B=(C), C=(D)
    looks_image = True
    for i, letter in enumerate(['A', 'B', 'C']):
        v = opts.get(letter, '')
        if v:
            v = str(v).strip()
            next_label = chr(ord('B') + i)  # A→(B), B→(C), C→(D)
            if not v.startswith(f'({next_label})'):
                looks_image = False
                break
    if looks_image:
        return True
    return False
```

#### 演進歷程

| 版本 | 邏輯 | 問題 |
|------|------|------|
| v1 | `len(empty) >= 2` | 只抓到 113 Q7、115 Q29（D 空字串的題目），其他題 fallback regex 填入了 `(B)/(C)/(D)` 偽文字 |
| v2 | 加 `re.fullmatch(r'\([B-D]\)', v)` | 未考慮 C 選項可能夾帶後續漏字（如 115 Q41 C=`(D)表(六)日期...`），`fullmatch` 太嚴格 |
| **v3（最終）** | `A.startswith('(B)'), B.startswith('(C)'), C.startswith('(D)')` | A=(B) 且 B=(C) 且 C=(D) 的典型 fallback 模式，使用 `startswith` 容忍 C 尾端漏字 |

### 4.2 核心函式 `mark_image_options()`

檔案：`pipeline/parse_questions.py:91-111`

```python
def mark_image_options(questions: list[dict], year: str) -> None:
    """Tag questions whose options are images with an image-options filename.

    Sets two fields on each in-place:
      - image_options: "{year}{q:02d}i.jpg" or None
      - image_options_full: True  → all of A/B/C/D are images
                            False → only A/B/C are images, D has text
                            None  → not an image-option question
    """
    for q in questions:
        if not has_image_options(q.get('options', {})):
            q['image_options'] = None
            q['image_options_full'] = None
            continue
        qnum = q['number']
        fname = f'{year}{qnum:02d}i.jpg'
        d_text = q.get('options', {}).get('D', '')
        d_empty = not d_text or not str(d_text).strip()
        q['image_options'] = fname
        q['image_options_full'] = bool(d_empty)
```

呼叫時機：`parse_questions()` 最後一行（`parse_questions.py:457`），在所有
題目解析、`clean_option_tail` 清洗完成後執行。

### 4.3 JSON Schema 新增欄位

```jsonc
{
  "number": 29,
  "type": "單題",
  "stem": "...",
  "options": {"A": "", "B": "", "C": "", "D": ""},
  "image_options": "11529i.jpg",    // 非 null → 圖片選項題
  "image_options_full": true,        // true=A~D全圖, false=A~C圖+D文字
  ...
}
```

- `image_options`：`null` 表示非圖片選項題，非空字串表示 `web/pic/item/` 下的裁切檔名
- `image_options_full`：僅在 `image_options` 非 null 時有意義
  - `true`：4 個字母按鈕 + 一張選項圖
  - `false`：3 個字母按鈕（A/B/C）+ 一張選項圖 + D 文字區塊

### 4.4 既有輔助函式 `clean_option_tail()`

檔案：`pipeline/parse_questions.py:41-55`

```python
def clean_option_tail(opt_text: str) -> str:
    """Remove trailing figure/table label fragments."""
    opt_text = re.sub(r'[圖表](?:\(\S*\))?$', '', opt_text)
    opt_text = re.sub(r'\d{1,2}$', '', opt_text)
    opt_text = re.sub(r'[\x00-\x1f\uFFFD]+$', '', opt_text)
    return opt_text.strip()
```

此函式在 `mark_image_options` 之前對每個選項呼叫，清除 PDF 文字流洩漏的
圖表標籤尾部和頁碼數字。對圖片選項題影響有限（圖片選項本來就無文字可清），
但對確保非圖片題品質是必要的。

### 5.1 整體架構

```

├── _build_page_map(year, pdf_path)

├── locate_option_bbox(page, qnum)
│     7 層定位策略找選項區 bbox（PDF 點座標）
├── crop_and_save(page_png, bbox, out_path, padding=10)
│     DPI 200, scale=200/72, JPG quality=90
└── main()
      逐題裁切 → web/pic/item/{year}{q:02d}i.jpg
```

### 5.2 `locate_option_bbox()` — 7 層 bbox 定位策略

這是整個裁切腳本的核心。輸入 PyMuPDF page object + 題號，輸出 `[x0, y0, x1, y1]`
四邊界（PDF points）。

```
Step 1:  掃描頁面所有文字 blocks，匹配 r'(\d{1,2})\.' 建立 qnum→y 對應
         去重：同一題號出現兩次 → 取 y 較大者（真正的題目起點）
Step 2:  在 [q.top, next_q.top] 區間內找 (A)(B)(C)(D) 標籤 → opt_top, opt_bottom
         （初始範圍 = 標籤 Y 區間 ± 邊距）
Step 3:  擴展至嵌入圖片（page.get_image_info()）
         條件：圖片落在選項垂直區間內（不超出上一題/下一題邊界）
         向上/向下延伸以包含圖片
Step 4:  擴展至繪圖 frame rects（page.get_drawings() → items[0]=='re'）
         篩選：w≥40, h≥20，在垂直區間內
Step 5:  擴展至繪圖 clusters（向量地圖等無 rect 的繪圖）
         收集所有非 rect 繪圖的 bbox → Y 排序 → 合併間距 ≤8pt 的 cluster
         → 取 w>60 或 h>40 的大型 cluster 延伸邊界
Step 6:  下一題邊界約束（不超過下一題題號 y）
Step 7:  X 邊界：全頁寬 - 左右 margin（40pt / 45pt）
```

#### 頁碼對應的演進

| 版本 | 方法 | 問題 |
|------|------|------|
| v1 | 硬編碼 `SINGLE_PAGES` dict（~4題/頁） | 113 Q7 猜頁 4 實際在頁 3；114 Q29 猜頁 10 實際在頁 8；115 年全部猜錯 |
| **v2（最終）** | `_build_page_map()` 自動掃 PDF 文字層 | 精準，支援快取、全形點號 fallback |

### 5.3 `crop_and_save()` — 裁切與儲存

```python
def crop_and_save(page_png: str, bbox: list[float], out_path: str, padding: int = 8) -> tuple | None:
    scale = DPI / 72                    # 200/72 ≈ 2.7778
    img = Image.open(page_png)
    crop = (
        max(0, int((bbox[0] - padding) * scale)),
        max(0, int((bbox[1] - padding) * scale)),
        min(img.width, int((bbox[2] + padding) * scale)),
        min(img.height, int((bbox[3] + padding) * scale)),
    )
    if crop[2] <= crop[0] or crop[3] <= crop[1]:
        return None                     # 無效裁切
    cropped = img.crop(crop).convert('RGB')
    cropped.save(out_path, 'JPEG', quality=90)
    return cropped.size
```

關鍵參數：
- **DPI = 200**：與 `assets/{year}/pages/` 渲染 DPI 一致
- **scale = DPI / 72**：PDF 點座標轉像素的倍率
- **padding = 10**：每邊額外 10pt 邊距（比一般圖表 crop 的 4pt 更寬鬆）
- **quality = 90**：JPG 品質
- **.convert('RGB')**：確保無 alpha 通道的純色 JPG

## 7. 過程中的困難與解決

### 7.1 困難 #1：Parser fallback regex 的偽文字汙染

**現象**：多數圖片選項題的 `options.A` = `(B)`、`options.B` = `(C)`、
`options.C` = `(D)`。parser 的 fallback regex `\(([A-D])\)\s*(.+?)` 在
選項文字為空時，會把下一選項標籤 (`(B)`) 當成 A 的內容抓進來。

這導致 `has_image_options()` v1 的 `len(empty) >= 2` 門檻抓不到這些題目——
因為選項並非空字串，而是被填入了假標籤。

**解決**：在 `has_image_options()` 加入 fallback-grab 檢測器：
檢查 `A.startswith('(B)') && B.startswith('(C)') && C.startswith('(D)')`。
使用 `startswith` 而非 `fullmatch` 以容忍 C 尾端夾帶後續漏字
（如 115 Q41 的 C = `(D)表(六)日期...`）。

### 7.2 困難 #2：頁碼對應不準

**現象**：初版使用硬編碼 `SINGLE_PAGES` 對應表（~4 題/頁），但實際年度的
題目密度不同：
- 112 年 Q7 在頁 3，114/115 年 Q7 在頁 2
- 115 年 Q27、Q29 都在頁 8（題目密度高）
- 115 年 Q41 在頁 11（不是 13）

結果 5/7 題找不到題號 → `locate_option_bbox` 失敗。

**解決**：改為 `_build_page_map()` 自動掃 PDF 文字層。對每個頁面（2-15），
用 `page.get_text("dict")` 掃描 blocks/lines，匹配 `r'^\s*(\d{1,2})\.\s+'`
及全形點號 `\uff0e` fallback，建立精確的 qnum→page 快取表。

### 7.3 困難 #3：Bbox 定位抓到別題的選項標籤

**現象**：`locate_option_bbox` v1 只找「題號之後」的第一組 (A)-(D)，但同一頁
上可能有多題（如 115 年頁 8 有 Q19-Q29 共 6 題）。`(D)` 標籤的 y 座標篩選
`y > q_y_end` 無法區分是第 N 題還是第 N+1 題的選項。

**解決**：先掃描頁面所有題號建立 `q_map`，找到當題題號 `q_top` 和下一題題號
`next_q_y`。選項標籤搜尋限制在 `[q_top, next_q_y]` 區間內。無法找到下一題時
放寬至 `[q_top, page_h - 10]`。

### 7.4 困難 #4：裁切高度不足（只切到文字標籤行）

**現象**：裁切 bbox 只有 30-124pt 高（只包含 (A)-(D) 文字行），不含下方的
實際選項圖片。

根因：
1. 嵌入圖片檢測只找「標籤下方」的圖，但有些選項圖在標籤上方或重疊
2. 向量地圖選項（如 114 Q29）無 `page.get_image_info()` 回傳
3. `Rect.y3` 屬性不存在（PyMuPDF Rect 是 `y0, y1`，不是 `y3`）

**解決**：
- 嵌入圖片檢測改為垂直區間重疊判定（不限於下方）
- 加入 frame rect 檢測（`items[0]=='re'`, w≥40, h≥20）
- 加入繪圖 cluster 檢測：收集非 rect 繪圖 bbox → Y 排序 → 合併 → 取大型 cluster
- 修正 `r.y3` → `r.y1`

### 7.5 困難 #5：題組子題 (114 Q43) 的作答狀態隔離

**現象**：`selectAnswer()` 用 `document.querySelectorAll('.q-option')` 清除
全頁面的 selected 狀態。圖片選項題不使用 `.q-option` class，而且題組子題
場景中同一頁可能有多個子題的選項區。

**解決**：新增 `selectImageAnswer(qnum, letter, el, container)`，只在傳入的
`container` 內清除 `.q-imgopt-btn, .q-imgopt-text-d`。`container` 對單題是
`#quiz` 的 optsDiv，對題組子題是 stemDiv 內的 opts 容器。

## 11. 技能化補充

以下為將本方案封裝為 skill / generalized script 時應注意的擴充點：

### 11.1 通用化裁切腳本

現有 `crop_image_options.py` 未接受命令列參數；通用化後應支援：

```bash
# General form
python pipeline/crop_image_options.py [--year 115] [--dry-run] [--padding 10]

# --year N: only process specific year
# --dry-run: print bbox without saving
# --padding N: override default 10pt padding
```

### 11.2 裁切邊界微調的影響因子

裁切結果的品質取決於下列參數，通用化時應設為可配置：

| 參數 | 預設值 | 影響 |
|------|-------|------|
| `margin_left` | 40pt | 左切邊。太小會切到頁碼；太大會切到選項 |
| `margin_right` | 45pt | 右切邊 |
| `padding` | 10pt | 四周邊距。增加可防止選項邊緣被切 |
| `opt_top_margin` | -10pt | 選項上方邊距 |
| `opt_bottom_initial` | +20pt | 初始下方邊距（文字標籤後） |
| `img_extend` | +10pt | 嵌入圖片後的延伸 |
| `frame_extend` | +10pt | frame rect 後的延伸 |
| `cluster_extend` | +15pt | 繪圖 cluster 後的延伸 |
| `cluster_min_size` | w>60 or h>40 | cluster 最小尺寸門檻 |
| `DPI` | 200 | 頁面渲染解析度 |
| `quality` | 90 | JPEG 品質 |

### 11.3 頁碼快取的跨年度相容性

`_build_page_map()` 假設題號在頁 1-14 範圍內且掃描 pages[1:15]。
若未來（116+）年度格式變動（如新增頁面、題號跨頁方式改變），
需調整 `min(15, doc.page_count)` 範圍與題號去重策略。

去重策略（取 y 較大者 = 較低位置的題號行）假設：
- 較高位置的同號文字是「前頁延續的題幹文字」
- 較低位置的才是該題的真正起點

此假設在極端跨頁場景（題號行在中間被分頁截斷）可能失效，
需在 `q_map` 建立時加入額外判斷（如比對 span 字級）。

### 11.4 向量地圖選項（無嵌入圖片的場景）

目前裁切依賴以下 4 種視覺元素：
1. 文字 (A)-(D) 標籤（必定存在）
2. 嵌入點陣圖（`get_image_info()`）
3. 繪圖 frame rect（`get_drawings() → 're' items`）
4. 繪圖 cluster（向量繪畫 → bbox 合併）

如果未來出現純向量且無 frame rect 的選項（如僅有 `l`/`c` 路徑的小型
圖形），cluster 合併可能失效（因為 `rect.width` 和 `rect.height` 對單一
路徑而言可能很小）。需加入 `qu`（quad）或 `l`（line）items 的篩選，
以及更細粒度的 cluster 合併策略。

### 11.5 `image_options_full` 的多來源驗證

`image_options_full` 目前僅由 `options.D` 是否為空字串判斷。
在 `crop_image_options.py` 的 dry-run 模式中，建議加入**頁面視覺
分析**（透過 opencode-see-image plugin 讀取頁面 PNG 的視覺描述
來確認 D 選項確實是文字或圖片），作為第二驗證來源。

### 11.6 跨題裁切邊界的競態

`locate_option_bbox` Step 6（下一題約束）在題目密集排列（如 115 年
頁 8 有 Q19、Q20、Q21、Q22、Q23、Q27、Q29 同時在該頁）時，
相鄰題目的邊界僅差數行文字。裁切結果可能比預期窄。

解法方向：
- 在 `next_q_y` 約束前，先確認 Step 3-5 的圖片/繪圖 cluster
  已完全被選項標籤 bbox 涵蓋
- 若圖片 cluster 明顯超出文字標籤區域但在 `next_q_y` 之前，
  優先使用圖片 cluster 邊界（目前 Step 6 會強制裁切到 `next_q_y - 5`）

### 11.7 `selectImageAnswer` 的作答狀態序列化

現有 `selectedAnswers` 物件（`{qnum: letter}`）已正確記錄圖片
選項題的作答。前端無需額外處理。但若未來加入**答案核對模式**
（顯示正確答案），需確保 `.q-imgopt-btn.correct` / `.q-imgopt-btn.wrong`
CSS 類別正確套用。

### 11.8 Skill 邊界

本方案適合作為 `jhexam-figure-crop` skill 的子功能或獨立 skill。
建議命名：`jhexam-image-option`。

Skill 內容應包含：
- 上述完整的 7 題清單 + 頁碼對應
- `locate_option_bbox()` 的 7 層策略說明
- 通用化 CLI 介面規格（§11.1）
- 參數調整指南（§11.2）
- 邊際案例處理（§11.4-11.6）
