# 社會科會考題庫 — 題組子題結構改良方案 V4

## 0. 摘要

國中教育會考社會科題本每年 54 題（Q1–Q42 單題、Q43/44–Q54 題組）。目前 parser 在題組子題上有五類結構性錯誤：

| # | 問題 | 嚴重度 | 影響範圍 |
|---|------|--------|---------|
| P1 | 選文（passage）混入末題 `options.D` | 🔴 嚴重 | 3年 × 3~4組 = 10 題 |
| P2 | 題組完全缺 passage 欄位 | 🔴 嚴重 | 所有 36 道題組子題 |
| P3 | 圖表歸屬錯位（passage 圖被歸到子題） | 🟡 中 | 含 passage 引用圖表的題組 |
| P4 | 表/圖標籤滲入末題選項尾巴 | 🟡 中 | 如 112 Q45 D 末尾多一個「表」字 |
| P5 | 圖片選項無法擷取（選項為圖，非文字） | 🟠 限制 | 114 Q43（選項 A/B/C 為空，缺 D） |

---

## 1. 根因分析

### 1.1 PDF 文字流結構

PyMuPDF 萃取文字的順序為：**子題在前，選文在後**。選文永遠出現在該 group 最後一個選項 `(D)` 之後。

```
閱讀下列選文，回答第 52 至 54 題：
52. [子題問句]
(A)... (B)... (C)... (D)...
53. [子題問句]
(A)... (B)... (C)... (D)...
54. [子題問句]
(A)... (B)... (C)...
(D) 以修法程序而言，皆可由行政機關自行修訂    ← 末題最後選項
美國的反體罰組織於1998年發起...                ← 整段選文 (passage)
圖(十六)                                       ← passage 引用的圖標籤
```

### 1.2 Parser 缺口鏈式效應

**第一重**：`parse_section()` 的 split regex `(?=\n\d{1,2}\.\s|\Z)` — 末題後無下一題號，body 經 `\Z` 吃到 block 結尾 → **passage 被吞入末題 body**。

**第二重**：`extract_question()` 的選項 regex `[^\(]+` — 只排除半形 `(`，中文全形 `（」「` 照吞 → **passage 整段成為 D 選項尾巴**。

### 1.3 中間夾層問題（V3 未識別）

> [!WARNING]
> 末題 `(D)` 與 passage 之間，有時夾有**不相關的頁面內容**（如同頁其他題的表格資料）。

以 112 年 Group 44-45 為例，raw text 順序為：

```
(D)日本藉由牡丹社事件擴張勢力     ← Q45 option D
表(八)                              ← ⚠ Q43 單題的表格標題（與本題組無關）
原因                                ← ⚠ Q43 表格內容
年齡(歲)
30-34  65.37  0.89 ...              ← ⚠ 表格數據（大量數字行）
歷史上的琉球王國與中國關係密切...     ← 真正的 passage 才開始
```

這導致：
- 112 Q45 的 D 選項尾巴多了一個「表」字：`日本藉由牡丹社事件擴張勢力表`
- `find_passage_start()` 必須能跳過中間的表格數據雜訊

### 1.4 控制字元

112 年部分頁面有 `\u0016`、`\u0019` 等控制字元混入，需清理。

### 1.5 圖片選項（P5）

114 年 Q43 的 A/B/C/D 選項是圖片而非文字，PDF 文字流中只有空白。這是 PyMuPDF 文字擷取的固有限制，無法在 parser 層面解決，需在前端以整頁圖片 fallback 處理。

## 3. 設計原則

| 原則 | V3 設計 | V4 修正 |
|------|---------|---------|
| schema 向後相容 | ✓ 保留現有欄位 | ✓ 不變 |
| passage 存放位置 | 僅首題存 passage | **所有同組子題都存 passage**（理由見 §3.1） |
| 圖表歸屬精準 | passage_figures 與 figures 分離 | ✓ 不變 |
| parser 雙重保險 | 切 passage + 強化 (D) 截斷 | **三重保險**：加入表/圖標籤清理 |
| find_passage_start | 4 層啟發式 | **重新設計**：考慮中間夾層（§4.4） |
| 前端三欄式 | CSS Grid + sticky | ✓ 不變，微調 |

### 3.1 為何改為所有子題都存 passage

V3 的「僅首題存 passage」設計有兩個問題：
1. **前端需額外查找**：顯示 Q53 時需從 JSON 中向前搜尋同 `group_id` 的首題取 passage，增加複雜度
2. **獨立性差**：每道題不能獨立使用，必須搭配同組首題才完整

V4 改為**每道題組子題都攜帶 passage**。雖然有冗餘（3 份相同字串），但 JSON 總量增加極少（每組 passage 約 200–500 字 × 3 份 ≈ 2KB），換取前端邏輯大幅簡化。

### 4.2 控制字元清理

```python
def clean_control_chars(text: str) -> str:
    """移除 PDF 萃取時混入的控制字元（\u0000-\u001f 不含 \n）"""
    return re.sub(r'[\x00-\x09\x0b-\x1f]', '', text)
```

在 `parse_questions()` 中 `raw = f.read()` 之後立刻呼叫。保留 `\n` (`\x0a`) 不刪。

### 4.3 重寫 `parse_group_section()`

```python
def parse_group_section(text: str, fig_page_map: dict) -> list[dict]:
    """
    完整流程：
    1. block_pattern 抓每個題組 block
    2. 對每個 block，先分離 passage 與 questions 區域
    3. 用乾淨的 questions 區域解析子題
    4. 末題選項尾巴清理（圖/表標籤）
    5. 圖表歸屬分離（passage_figures vs figures）
    6. passage 寫入所有子題
    """
    block_pattern = re.compile(
        r'閱讀下列選文，回答第\s*(\d+)\s*至\s*(\d+)\s*題[：:]?\s*\n'
        r'(.*?)(?=閱讀下列選文|試題結束|\Z)',
        re.DOTALL
    )
    blocks = block_pattern.findall(text)
    questions = []

    for q_start_s, q_end_s, content in blocks:
        q_start, q_end = int(q_start_s), int(q_end_s)
        group_id = f"{q_start}-{q_end}"

        # --- 步驟 2：分離 passage 與 questions ---
        passage, clean_content = split_passage_and_questions(
            content, q_start, q_end
        )

        # --- 步驟 3：用 clean_content 解析子題 ---
        sub_qs = parse_section(clean_content, '題組子題')
        sub_qs.sort(key=lambda x: x['number'])

        # --- 步驟 4：末題選項尾巴清理 ---
        for q in sub_qs:
            for letter in list(q['options'].keys()):
                q['options'][letter] = clean_option_tail(
                    q['options'][letter]
                )

        # --- 步驟 5：圖表歸屬 ---
        passage_figs = list(set(re.findall(
            r'[圖表]\([一二三四五六七八九十\d]+\)', passage
        )))
        passage_fig_pages = {
            f: fig_page_map.get(f) for f in passage_figs
            if fig_page_map.get(f) is not None
        }

        for q in sub_qs:
            # 子題 figures 移掉 passage 已領的圖
            q['figures'] = [
                f for f in q.get('figures', [])
                if f not in passage_figs
            ]
            q['group_range'] = [q_start, q_end]
            q['group_id'] = group_id
            q['type'] = '題組子題'

        # --- 步驟 6：passage 寫入所有子題 ---
        for q in sub_qs:
            q['passage'] = passage if passage else None
            q['passage_figures'] = passage_figs
            q['passage_figure_pages'] = passage_fig_pages

        questions += sub_qs

    return questions
```

### 4.4 `split_passage_and_questions()` — 全新設計

> [!IMPORTANT]
> V3 的 `find_passage_start()` 假設末題 `(D)` 之後就是 passage，但實測發現中間可能夾有不相關的表格資料（如 112 Group 44-45）。V4 改用雙向搜索策略，在整個 block content 層級操作。

```python
def split_passage_and_questions(
    content: str, q_start: int, q_end: int
) -> tuple[str, str]:
    """
    將題組 block content 分離為 passage 和 questions 部分。

    策略：
    S1: 找最後一個 (D) 選項的結束位置
    S2: 在 (D) 之後的文字中，跳過雜訊找到 passage 起點
    S3: 提取 passage 後，questions = content 扣除 passage
    """
    # 找 content 中最後一個 (D) 及其後的文字
    d_positions = [m.end() for m in re.finditer(r'\(D\)', content)]
    if not d_positions:
        return '', content

    last_d_end = d_positions[-1]
    after_d = content[last_d_end:]

    # 找 passage 起點
    passage_start_in_after = find_passage_boundary(after_d)

    if passage_start_in_after is None:
        return '', content

    abs_passage_start = last_d_end + passage_start_in_after
    passage = content[abs_passage_start:].strip()
    clean_content = content[:abs_passage_start].strip()

    return passage, clean_content


def find_passage_boundary(text_after_d: str) -> int | None:
    """
    在 (D) 選項之後的文字中，找到 passage 的起始位置。

    需要跳過：
    1. (D) 選項本身的文字（通常一行）
    2. 可能的表格/圖標籤及數據行

    識別 passage 起點：連續中文敘述句（≥10 中文字元）
    """
    lines = text_after_d.split('\n')

    # D 選項文字在第一行
    option_d_end = 0
    for i, line in enumerate(lines):
        stripped = line.strip()
        if not stripped:
            continue
        if i == 0:
            option_d_end = i + 1
            continue
        break

    cumulative_pos = 0
    for i, line in enumerate(lines):
        if i < option_d_end:
            cumulative_pos += len(line) + 1
            continue

        stripped = line.strip()

        # 空行：跳過
        if not stripped:
            cumulative_pos += len(line) + 1
            continue

        # 表/圖標籤行：跳過
        if re.match(r'^[圖表]\([一二三四五六七八九十\d]+\)$', stripped):
            cumulative_pos += len(line) + 1
            continue

        # 純數字/表格數據行：跳過
        if re.match(r'^[\d\s,.\-%]+$', stripped):
            cumulative_pos += len(line) + 1
            continue

        # 很短的行（< 6 字且非中文段落）：跳過
        if len(stripped) < 6 and not re.search(
            r'[\u4e00-\u9fff]{6,}', stripped
        ):
            cumulative_pos += len(line) + 1
            continue

        # 頁碼/頁面標記：跳過
        if re.match(
            r'^(=== PAGE \d+ ===|\d{1,2}|請翻頁繼續作答|試題結束)$',
            stripped
        ):
            cumulative_pos += len(line) + 1
            continue

        # 到此：可能是 passage 開頭
        cjk_count = len(re.findall(r'[\u4e00-\u9fff]', stripped))
        if cjk_count >= 10:
            return cumulative_pos

        cumulative_pos += len(line) + 1

    return None
```

### 4.5 `clean_option_tail()` — 新增

```python
def clean_option_tail(opt_text: str) -> str:
    """
    清除選項文字尾部殘留的圖/表標籤碎片。
    例如：「日本藉由牡丹社事件擴張勢力表」→ 去掉末尾的「表」
    """
    # 移除末尾的孤立「圖」「表」字
    opt_text = re.sub(r'[圖表]$', '', opt_text)
    # 移除末尾的頁碼數字（如 passage 後的頁碼 ...13）
    opt_text = re.sub(r'\d{1,2}$', '', opt_text)
    return opt_text.strip()
```

> [!WARNING]
> `clean_option_tail()` 有誤殺風險——如果某選項本來就以「表」或數字結尾，會被錯誤刪除。需搭配長度判斷：只在 passage 已分離後的殘留清理時啟用。

### 4.6 `extract_question()` 強化

```python
def extract_question(qnum: int, body: str, qtype: str) -> dict | None:
    """強化版：選項 regex 改用結構邊界，二次清理控制字元。"""

    body = re.sub(r'[\x00-\x09\x0b-\x1f]', '', body)

    opt_a = body.find('(A)')
    if opt_a == -1:
        return None

    stem = body[:opt_a].strip()
    stem = re.sub(r'\s+', '', stem)

    options_raw = body[opt_a:]

    opts = {}
    for letter in ['A', 'B', 'C', 'D']:
        if letter == 'D':
            pat = rf'\({letter}\)\s*(.+?)$'
        else:
            next_letter = chr(ord(letter) + 1)
            pat = rf'\({letter}\)\s*(.+?)(?=\s*\({next_letter}\)|\Z)'
        m = re.search(pat, options_raw, re.DOTALL)
        if m:
            opts[letter] = re.sub(r'\s+', '', m.group(1).strip())

    # Fallback
    if len(opts) < 3:
        for letter in ['A', 'B', 'C', 'D']:
            if letter not in opts:
                m = re.search(
                    rf'\({letter}\)\s*(.+?)(?=\([A-D]\)|\Z)',
                    options_raw, re.DOTALL
                )
                if m:
                    opts[letter] = re.sub(r'\s+', '', m.group(1).strip())

    all_text = stem + ''.join(opts.values())
    figs = list(set(re.findall(
        r'[圖表]\([一二三四五六七八九十\d]+\)', all_text
    )))

    return {
        'number': qnum,
        'type': qtype,
        'group_range': None,
        'group_id': None,
        'stem': stem,
        'options': opts,
        'figures': figs,
        'passage': None,
        'passage_figures': [],
        'passage_figure_pages': {},
    }
```

### 6.5 app.js 邏輯骨架

```javascript
function renderQuestion(q) {
  if (q.type === '題組子題') {
    return renderGroupQuestion(q);
  }
  return renderSingleQuestion(q);  // 現有 layout 不變
}

function renderGroupQuestion(q) {
  // V4 簡化：passage 直接從 q.passage 取得

  const groupQs = getAllQuestions().filter(
    item => item.group_id === q.group_id
  );

  // 判斷是否需要重建（跨 group）
  const currentGroupEl = document.querySelector(
    `.group-question[data-group-id="${q.group_id}"]`
  );

  if (currentGroupEl) {
    // 同 group 切題：只更新右下題目區
    updateStemArea(currentGroupEl, q, groupQs);
  } else {
    // 跨 group 或首次：完整重建
    buildGroupDOM(q, groupQs);
  }
}
```
