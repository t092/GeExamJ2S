# 圖片（圖）處理總結

## 基本原理

國中教育會考社會科題本中，圖形標籤 `圖(N)` 位於圖形**下方**，裁切方向為**往上**搜尋圖形邊界。

## 四階段偵測流程（依優先序）

### 1. 框架偵測（frame）— 最優先
- 使用 `page.get_drawings()` 提取所有向量矩形（`item[0] == "re"`）
- 篩選條件：`width > 80` 且 `height > 50`
- 去重：bbox 四個值誤差 <5pt 的視為重複
- 匹配邏輯：圖形框的**底部**必須在標籤**上方**且靠近
  ```
  rb[3] <= ly + 20          # 框底在標籤上方 ≤20pt
  rb[1] >= ly - 500         # 框頂在標籤上方 ≤500pt
  rb[0] <= lx + 80          # x 軸對齊
  rb[2] >= lx - 80
  ```
- 裁切範圍：`[框左, 框頂, page_w - 55, ly + 15]`
  - **右界統一使用 `page_w - 55`**，而非原始框架右界。原因：PDF 中部分圖形框架右界過窄（如地圖的經緯度標記超出框架），強制擴展到頁面右邊界可避免裁切

### 2. 嵌入式圖片偵測（embedded）
- 使用 `page.get_image_info()` 提取嵌入的點陣圖
- 篩選條件：`width > 80` 且 `height > 50`
- 匹配邏輯與 frame 相同
- 裁切範圍：`[圖左, 圖頂, 圖右, ly + 15]`

### 3. 繪圖元素叢集（cluster）
- 適用於由大量小繪圖元素組成的圖形（如報紙文字、地圖細節）
- 篩選條件：`r.width > 5 and r.height > 5 and r.x0 > 310`
- 搜尋範圍：標籤上方 `ly - 300` 到 `ly + 10`
- 叢集演算法：y 軸差距 ≤10pt 且 x 軸重疊 → 合併為同一叢集
- 篩選合格叢集：`width > 50, height > 50, len > 10`
- 裁切範圍：`[叢集左, 叢集頂, 叢集右, ly + 15]`

### 4. 啟發式裁切（crop）— 最後手段
- 適用於前三種方法都找不到的圖形
- 上邊界 `fig_top`：
  - 預設 `max(ly - 200, 50)`（圖形最多在標籤上方 200pt）
  - 若上方有其他 `圖/表` 標籤，且其位置比 `ly - 200` 更近，則使用該標籤位置
  - 檢查右欄（x>310）是否有繪圖元素，找出最上方者，tighten `fig_top`
- 左邊界：
  - 若左欄（x≤310）有繪圖元素 → 使用 `x=66`（全幅）
  - 否則 → 使用 `x=300`（右欄）
- 裁切範圍：`[left_x, fig_top, page_w - 55, ly + 15]`

## 優先序與選擇邏輯

方法優先序：`frame(0) > embedded(1) > cluster(2) > crop(3)`

同優先序時的選擇規則（在 `process_labels` 中實現）：
1. **面積優先**：若同階方法產生的面積大於現有最佳者的 1.5 倍 → 取代
2. **高度優先**：否則，選擇高度較小者（更緊湊的裁切）

額外過濾：
- **小型 frame/embedded 捨棄**：若 frame 或 embedded 結果的寬度小於 150pt，捨棄該結果，交由 crop fallback 處理。原因：部分圖形的向量框架只涵蓋圖例框而非完整圖形（如火山地圖的圖例框 w=113pt），crop fallback 能產生更完整的裁切

## 關鍵參數

| 參數 | 值 | 說明 |
|------|-----|------|
| DPI | 200 | 頁面渲染解析度 |
| 裁切 padding | 4pt | 四周額外邊距 |
| 圖形框最大高度 | 400pt | 過濾異常大的框 |
| fig_top 預設 | `ly - 200` | 標籤上方搜尋範圍 |
| frame 右界 | `page_w - 55` | 強制擴展到頁面右邊界 |
| frame/emb 最小寬度 | 150pt | 小於此值捨棄 |

## 輸出格式
- 檔名：`{year}p{NN:02d}.jpg`（如 `114p01.jpg`）
- JSON 記錄：`web/pic/figures_{year}.json`

## 注意事項
- 圖形偵測是**完全獨立**的，不與表格偵測共用任何參數
- `detect_crop_fig` 的 fig_top 使用 `max(ly - 200, 50)` 強制下限，避免納入不相關的題目內容
- 框架偵測的 `rb[3] <= ly + 20` 確保只匹配標籤緊鄰上方的框架
- 框架右界不使用原始 bbox 右界，統一用 `page_w - 55`，解決部分圖形（地圖、圖表）邊界標記超出原始框架的問題
- 圖形可能由多個元件組成（如地圖本體 + 圖例框），frame detection 可能只抓到圖例框。需靠小型結果過濾機制讓 crop fallback 接手
