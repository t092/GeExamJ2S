# 製作題組技巧 — 從 PDF 考題萃取題組資料的實戰經驗

> 本文記錄 JHexam 專案中處理國中教育會考社會科「題組子題」的完整過程，
> 涵蓋問題診斷、解法設計、邊際案例與多年份驗證，作為日後製作題組
> skill 的知識基底。

---

## 1. 核心場景

會考題本每年 54 題：Q1–Q42 單題，Q43–Q54 為 4 個題組（每個 3 題）。
每個題組由一段「閱讀選文」（passage）和三道子題構成。

### 1.1 單題 vs 題組的根本差異

| 面向 | 單題 | 題組子題 |
|------|------|---------|
| 題幹位置 | 題目即題幹 | 共享選文（passage）獨立於子題之外 |
| 圖表歸屬 | 全屬該題 | 區分 passage 圖 vs 子題圖 |
| JSON 結構 | 平面欄位 | 需要 group_id + passage 欄位 |
| 前端排版 | 題目 + 選項 | 三區塊（選文 / 圖 / 題目） |

### 1.2 目標輸出結構

```jsonc
// 每個題組子題攜帶完整 passage 資訊（V4 設計）
{
  "number": 52,
  "type": "題組子題",
  "group_range": [52, 54],
  "group_id": "52-54",          // 同組共用 ID
  "stem": "子題問句",
  "options": {"A":"...", "B":"...", "C":"...", "D":"..."},
  "passage": "美國的反體罰組織於1998年發起...",   // 閱讀選文
  "passage_figures": ["圖(十六)"],               // 選文自身的圖表
  "passage_figure_pages": {"圖(十六)": 15},
  "figures": [],                 // 子題自身的圖表（不含 passage 圖）
  "figure_pages": {}
}
```

## 2. 問題鏈：為什麼選文會跑到選項 D 裡？

### 2.1 PDF 文字流的實際順序

PyMuPDF 萃取文字後，題組 block 的順序永遠是「子題在前，選文在後」：

```
閱讀下列選文，回答第 52 至 54 題：
52. [子題問句]
(A)... (B)... (C)... (D)...
53. [子題問句]
(A)... (B)... (C)... (D)...
54. [子題問句]
(A)... (B)... (C)...
(D) 以修法程序而言，皆可由行政機關自行修訂    ← 末題最後選項
美國的反體罰組織於1998年發起...                ← 整段選文在 (D) 之後
圖(十六)                                       ← passage 引用的圖標籤
```

### 2.2 Parser 的雙重缺口

**第一重**：`parse_section()` 的題號 split regex 用 `\Z` 作為末題邊界。
因為 Q54 後面沒有 Q55，末題 body 一路吃到 block 結尾 → 選文被吞入末題 body。

**第二重**：`extract_question()` 的選項 regex 只排除半形 `(`，
但中文全形 `（）「」` 照吞不誤 → 選文整段成為 (D) 選項的尾巴。

### 2.3 中間夾層（常見但容易被忽略）

末題 (D) 與 passage 之間，有時夾有**不相關的頁面內容**，
例如同頁其他題目的表格數據。範例（112 Q44-45）：

```
(D) 日本藉由牡丹社事件擴張勢力     ← Q45 選項 D
表(八)                              ← Q43 單題的表格（與本題組無關）
原因
年齡(歲)  需照顧未滿12歲子女  ...    ← 表格數據（15+ 行純數字）
歷史上的琉球王國與中國關係密切...    ← 真正的 passage 在此
```

這層中間雜訊是 passage 切分最棘手的部分。

### 2.4 控制字元汙染

部分年度（如 112 年）的 PDF 文字流夾帶 `\x00`–`\x1f` 範圍的控制字元，
會在選項文字中殘留，干擾長度判斷。必須在所有文本處理前先清理。

## 3. 解法架構（V4 最終方案）

### 3.1 核心策略：先分離，再解析

```
Block Content
    │
    ├─ split_passage_and_questions()
    │       │
    │       ├─ 找最後一個 (D) → find_passage_boundary() 跳過中間雜訊
    │       │      → 策略 A：選文在題目之後（最常見）
    │       │
    │       └─ 若策略 A 無結果，找第一題之前的中文段落
    │              → 策略 B：選文在題目之前（如 113 Q44-45）
    │
    ├─ passage (乾淨選文)
    └─ clean_content (只有題目內容)
            │
            └─ parse_section() → extract_question()
                   │
                   └─ clean_option_tail() 清理選項尾巴
```

### 3.2 關鍵函式設計

#### `split_passage_and_questions(content, q_start, q_end)` → (passage, clean_content)

兩階段策略，互為 fallback：

```
策略 A（選文在後）：
  1. 找出 block 內最後一個 (D) 的位置
  2. 從 (D) 之後的文字呼叫 find_passage_boundary()
  3. 若找到 → passage = 邊界之後的文字，clean = 邊界之前

策略 B（選文在前）：
  1. 找出第一道子題題號在 content 中的位置
  2. 第一題之前的內容清除頁碼/空行後若 > 20 字 → 視為 passage
  3. clean = 第一題之後的內容
```

#### `find_passage_boundary(text_after_d)` → int | None

逐行掃描 (D) 之後的文字，**跳過**雜訊直到找到真正的 passage 開頭。
跳過規則（按優先級）：

| 規則 | 匹配模式 | 範例 |
|------|---------|------|
| 表/圖標籤行 | `^[圖表]\([一二三四五六七八九十\d]+\)$` | `表(八)` |
| 純數字/表格數據行 | `^[\d\s,.\-/%☆\u2605\u2606★☆\u3000]+$` | `30-34  65.37  0.89` |
| 頁碼/樣板文字 | `^(=== PAGE \d+ ===\|\d{1,2}\|請翻頁繼續作答\|試題結束)$` | `12` |
| 極短行（<6 字且 CJK <6） | `len < 6 and cjk_count < 6` | `原因`、`年齡(歲)` |
| 判斷為 passage 開頭 | `cjk_count >= 10` | `歷史上的琉球王國與中國...` |

> [!IMPORTANT]
> CJK 門檻 `>= 10` 是關鍵參數。太低會被短標題誤判，
> 太高可能跳過真正的 passage 段落（如果選文第一段恰好是短句）。

#### `clean_option_tail(opt_text)` → str

解析後的選項尾部清理，處理三類殘留：

```
1. 孤立的圖/表標籤尾部： 嚴格限制人口移入表 → 嚴格限制人口移入
2. 末尾頁碼數字：        由行政院院長任命12  → 由行政院院長任命
3. 控制字元殘留：        任何 \x00-\x1f      → 刪除
```

#### `clean_control_chars(text)` → str

在 `parse_questions()` 的第一道預處理步驟中執行，
移除 `\x00-\x09` 和 `\x0b-\x1f`，保留 `\x0a`（換行）。

### 3.3 圖表歸屬規則

| 圖表 label 出現位置 | 歸屬 | 原因 |
|--------------------|------|------|
| 僅在 passage 文字中 | `passage_figures` | passage 自身的插圖 |
| 僅在子題 stem | 該子題 `figures` | 子題引用的輔助圖 |
| 同時在 passage 與子題 | `passage_figures`（優先） | 避免重複，passage 圖層級更高 |

### 3.4 單題不變原則

所有新增欄位（`group_id`, `passage`, `passage_figures`, `passage_figure_pages`）
對單題 Q1–Q42 一律設為 `null` / `[]` / `{}`。

`clean_option_tail` 對單題也執行，但只清理明確的尾部汙染（孤立的圖/表字元、
頁碼數字），不影響正常選項文字。

## 4. 邊際案例清單

| # | 案例 | 出現年度 | 處理方式 |
|---|------|---------|---------|
| 1 | 選文在子題之後 | 112 Q46-48/49-51/52-54, 113 Q46-48/49-51/52-54, 114 全部 | 策略 A |
| 2 | 選文在子題之前 | 113 Q44-45 | 策略 B |
| 3 | 選文與末題 (D) 間夾表格數據 | 112 Q44-45 | `find_passage_boundary` 多層 skip |
| 4 | 選文跨頁、殘留頁碼 | 各年度 | `clean_control_chars` + regex 清理 |
| 5 | 選文無任何圖表 | 113 Q44-45, 113 Q46-48, 113 Q52-54, 114 Q46-48 | `passage_figures` 為空陣列 |
| 6 | 選項為圖片非文字 | 114 Q43（A/B/C 空白、D 缺失） | JSON 保留空字串，前端 fallback 整頁圖 |
| 7 | 選文圖被子題 stem 同時引用 | 112 Q52-54（圖(十六)） | passage 優先領圖，子題 `figures` 移除 |
| 8 | 控制字元混入選項尾 | 112 多題（Q20, Q28 等） | `clean_control_chars` 全量清理 |
| 9 | 表/圖標籤尾部滲入選項 | 所有年度單題 | `clean_option_tail`（只清孤立 圖/表） |

## 5. 開發流程建議

### 5.1 診斷階段

1. 先取一個題組的 raw text，直接肉眼讀，確認 PDF 文字流順序
2. 確認選文在題目前還是題目後（決定策略 A/B 的主次順序）
3. 檢查 (D) 與 passage 之間有無中間雜訊
4. 用 `repr()` 打印選項內容，檢查是否夾帶控制字元

### 5.2 實作階段

1. **控制字元清理**必須是第一道工序，放在所有 regex 之前
2. **passage 分離**必須在子題解析之前，確保子題 body 乾淨
3. 選項 regex 優先用 `[^\(]+`（舊法，已驗證穩定）
4. 末題 (D) 增加 `\n\d{1,2}\.\s` 邊界作為雙重保險
5. 圖表歸屬一定在 passage 分離後再做，避免重複

### 5.3 驗證階段

核心檢查項：

| # | 檢查 | 方法 |
|---|------|------|
| 1 | 題數無缺漏 | `len(questions) == 54` |
| 2 | 末題 D 不含選文 | 所有 group 末題 `options.D` 長度 ≤ 40 |
| 3 | group_id 一致 | 同 group 內子題 `group_id` 相同 |
| 4 | passage 完整 | 所有題組子題 `passage` 非 None、> 30 字 |
| 5 | passage 同組一致 | 同 `group_id` 內子題 `passage` 完全相同 |
| 6 | passage_figures 存在於 passage 文字 | 每個圖表 label 用 `in` 檢查 |
| 7 | 子題 figures 不含 passage 圖 | `set(figures) & set(passage_figures) == set()` |
| 8 | 單題不變 | Q1–Q42 的 stem/options/figures 與備份比對一致 |
| 9 | 控制字元已清除 | 所有欄位不含 `[\x00-\x09\x0b-\x1f]` |
| 10 | 選項尾巴乾淨 | 無選項以孤立「圖」「表」結尾 |
| 11 | 圖片選項保持空白 | 114 Q43 的空選項不捏造內容 |

> 建議將檢查項寫成自動化腳本（如 `check_groups.py`），每次 parse 完畢後執行。

### 5.4 前端整合

題組子題的前端排版與單題完全不同，需要專用版型：

- **三區塊結構**：上 passage（可收合）→ 下左圖（sticky）→ 下右題目
- **題組內切換**：同 group 子題切換時不重建 passage 區塊，只更新題目
- **無圖退化**：passage 無圖時右下題目跨全寬
- **響應式**：窄螢幕時三區塊垂直堆疊

## 6. 常見錯誤與對策

| 錯誤 | 原因 | 對策 |
|------|------|------|
| passage 長度為 0 | 只用了策略 A，沒處理「選文在前」的情況 | 加入策略 B fallback |
| passage 切點偏移 | `find_passage_boundary` 跳過規則不夠 | 逐行除錯，補齊 skip pattern |
| 選項 D 仍含長文 | passage 分離成功但 extract_question 沒截斷 | 雙重保險：分離 + D 邊界 regex |
| 單題選項被誤清 | `clean_option_tail` 過於激進 | 只清理孤立圖/表 + 頁碼；保留長度 > 長度的正常內容 |
| 圖表雙重歸屬 | passage 圖被子題 figures 同時引用 | passage 優先領圖，子題 figures 減去 |
| 跨頁 passage 斷裂 | PyMuPDF 跨頁插入 `=== PAGE N ===` | `parse_questions()` 前處理已移除頁碼樣板 |

## 7. 技術選型備忘

| 決策 | 選擇 | 理由 |
|------|------|------|
| passage 存放 | 每題都存（不存首題） | 前端邏輯簡化，總冗餘量極小（≤ 2KB） |
| (D) 選項 regex | `[^\(]+`（舊法） | 已驗證穩定；配套 passage 預先分離即可 |
| passage 邊界偵測 | 逐行 skip + CJK ≥ 10 門檻 | 比 state machine 簡單，比純 regex 穩健 |
| 圖表歸屬 | passage 優先領圖 | 避免同一 label 出現在兩處 |
| 控制字元清理 | 全域前處理 + 各函式二次清理 | 雙重安全網 |
| 驗證 | 自動化 11 項檢查 + 人工抽查 | 自動覆蓋結構面，人工覆蓋內容面 |

## 8. 適用場景

本技巧適用於以下 PDF 結構：

- 試卷類 PDF：題目在前、共用內容（選文/圖表）在後
- 課本/講義：文本段落 + 多道練習題
- 閱讀測驗：長文 + 多題選擇
- 任何需要從 PDF 中「分離共用本文與子題」的場景

不適用於：
- 選文與子題在 PDF 中完全交錯排列（需視覺版面分析）
- 多欄報紙式排版（需 `page.get_text("dict")` + bbox 分析）
