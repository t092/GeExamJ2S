# 社會科會考題庫 — 題組子題結構改良方案 V3（融合版）

## 0. 摘要

國中教育會考社會科題本每年共 54 題，Q1–Q42 為單題、Q43–Q54 為題組。目前 parser 在題組子題上有三類結構性錯誤：

| # | 問題 | 影響 |
|---|------|------|
| P1 | 選文（passage）混入末題 `options.D` | 所有 group 最後一題的 (D) 選項尾巴夾帶整段選文 |
| P2 | 題組完全缺 passage 欄位 | 前端無法顯示閱讀選文，考生無法作答題組 |
| P3 | 圖表歸屬錯位 | passage 引用的圖（如 圖(十六)）綁在子題 stem 而非 passage |

**受影響範圍**：三年共 12 個 group 的最後一題（112: Q45/Q48/Q51/Q54；113: Q48/Q51/Q54；114: Q45/Q48/Q51/Q54）。

本文融合兩份前置分析（`題組修改.md` 的 UX 設計 + `題組處理建議.md` 的工程穩健性），提出完整的 V3 改良方案。

---

## 1. 根因分析

### 1.1 PDF 文字流結構

PyMuPDF 萃取出的文字中，題組 block 的順序為：**子題在前，選文在後**。

```
閱讀下列選文，回答第 52 至 54 題：
52. 關於文中活動理念對臺灣帶來影響的過程...？
(A) 法律修改程序的變化
(B) 科技發展帶來的衝擊
(C) 風俗習慣的文化傳承
(D) 全球化下的文化交流
53. 根據圖(十六)判斷...？
(A)... (B)... (C)... (D)...
54. 關於文末前、後所提及我國的二項法規...？
(A)... (B)... (C)...
(D) 以修法程序而言，皆可由行政機關自行修訂
美國的反體罰組織於1998年發起「不打小孩日」的活動...  ← 整段選文
圖(十六)
```

### 1.2 Parser 雙重缺口（鏈式效應）

**第一重**：`parse_section` (L49–65) 的 split regex：
```
r'(?:^|\n)(\d{1,2})\.(?:\s*\n|\s+)(.*?)(?=\n\d{1,2}\.\s|\Z)'
```
末題 Q54 後面沒有下一題題號（`\n\d{1,2}\.\s`），於是 body 經 `\Z` 一路吃到 block 結尾 → **選文整段被吞入 Q54 的 body**。

**第二重**：`extract_question` (L108) 的選項 regex：
```python
re.findall(r'\(([A-D])\)\s*([^\(]+)', options_raw)
```
`[^\(]+` 只排除半形 `(`，中文全形 `（）` `「」` 照吞不誤 → 選文整段變成 Q54 選項 D 的尾巴。

### 1.3 額外問題：控制字元

112 年部分頁面（Page 13 等）有數字控制字元（`\u0016`、`\u0019` 等）混入文字流，可能干擾 passage 切點判斷。需要在 parser 中加入清理邏輯。

### 1.4 圖表歸屬錯位

passage 中的圖表（如 Q52–54 的 圖(十六)）實際出現於選文末尾，不屬於任何子題。目前 parser 將所有 `[圖表]\(...)` label 歸到子題，導致 Q53 `figures` 含有 `"圖(十六)"`。

---

## 2. 設計原則

| 原則 | 說明 |
|------|------|
| **schema 向後相容** | 保留現有全部欄位，新增欄位舊前端可忽略 |
| **passage 僅存首題** | 同一 group 只在第一子題存 passage，避免 3 倍冗餘 |
| **圖表歸屬精準** | passage 圖 (`passage_figures`) 與子題圖 (`figures`) 分離 |
| **parser 雙重保險** | 切 passage + 強化 (D) 截斷，兩層防護 |
| **前端三欄式 + UX** | CSS Grid 為主結構，疊加 sticky 圖區、passage 收合、題組內導覽 |
| **控制字元清理** | 比對選文前預先清理 `\u0000`–`\u001F` 等 |

## 3. Schema 改良

### 3.1 欄位定義

| 欄位 | 型別 | 位置 | 說明 |
|------|------|------|------|
| `number` | int | 所有題目 | 題號，1–54 |
| `type` | string | 所有題目 | `"單題"` 或 `"題組子題"` |
| `group_range` | [int, int] \| null | 題組子題 | `[52, 54]`（保留，向下相容） |
| `group_id` | string \| null | 題組子題 | `"52-54"`，同 group 內一致（**新增**） |
| `stem` | string | 所有題目 | 子題問句（單題則為完整題幹） |
| `options` | {A, B, C, D: string} | 所有題目 | 修正後不含選文 |
| `passage` | string \| null | 僅 group 首題 | 整段閱讀選文（**新增**） |
| `passage_figures` | string[] \| null | 僅 group 首題 | passage 引用的圖表 label（**新增**） |
| `passage_figure_pages` | object \| null | 僅 group 首題 | passage 圖 → 頁碼（**新增**） |
| `figures` | string[] | 所有題目 | 子題自身引用的圖（不含 passage 圖） |
| `figure_pages` | object | 所有題目 | 子題圖 → 頁碼 |

- 單題 (Q1–Q42)：新增欄位設為 `null` / `[]` / `{}`
- `group_id` 從 `group_range` 生成（`f"{r[0]}-{r[1]}"`），兩者並存
- `.passage` 僅 group 首題非 null；前端取 passage 時依 `group_id` 查首題

## 4. Parser 改良

### 4.1 修改總覽

| # | 函式 / 範圍 | 變更 |
|---|-----------|------|
| ① | `parse_questions()` 前處理 | 新增控制字元清理 `clean_control_chars()` |
| ② | `parse_group_section()` | 重寫：兩段式切 passage、末題截斷、圖表歸屬、寫入首題 |
| ③ | `extract_question()` | 強化選項 regex 為結構邊界、新增 `body_limit` 參數 |
| ④ | `find_passage_start()` | 新增輔助函式：定位 (D) 選項後的 passage 起點 |
| ⑤ | `resolve_figure_pages()` | 從 `data/<year>_figures.json` 查圖→頁碼 |

### 4.2 控制字元清理

```python
def clean_control_chars(text: str) -> str:
    """移除 PDF 萃取時混入的控制字元（\u0000-\u001f 不含 \n）"""
    return re.sub(r'[\x00-\x09\x0b-\x1f]', '', text)
```

在 `parse_questions()` 中，`raw = f.read()` 之後立刻呼叫。保留 `\n` (`\x0a`) 不刪。

### 4.3 重寫 `parse_group_section()`

```python
def parse_group_section(text: str) -> list[dict]:
    """
    完整流程：
    1. 用 block_pattern 抓每個題組（閱讀下列選文…至下一題組或檔尾）
    2. 對每個 block content 先切子題
    3. 從 content 擷取 passage（兩段式：首題之前 + 末題之後）
    4. 對末題重新渲染（截到 passage_start），確保 (D) 乾淨
    5. 圖表歸屬分離（passage_figures vs figures）
    6. passage 寫入首題
    """
    block_pattern = re.compile(
        r'閱讀下列選文，回答第\s*(\d+)\s*至\s*(\d+)\s*題[：:]?\s*\n(.*?)(?=閱讀下列選文|試題結束|\Z)',
        re.DOTALL
    )
    blocks = block_pattern.findall(text)
    questions = []

    for q_start, q_end, content in blocks:
        q_start, q_end = int(q_start), int(q_end)
        group_id = f"{q_start}-{q_end}"

        # --- 步驟 2：切子題 ---
        sub_qs = parse_section(content, '題組子題')
        sub_qs.sort(key=lambda x: x['number'])

        # --- 步驟 3：切 passage ---
        # passage_head：content 開頭到第一個子題之前（罕見但覆蓋）
        first_q_idx = find_q_start_index(content, sub_qs[0]['number'])
        passage_head = content[:first_q_idx].strip()
        # 移除 block_pattern 殘留的引導句（如仍殘留有 "閱讀下列選文…" 則清掉）
        passage_head = re.sub(r'^閱讀下列選文.*?\n', '', passage_head)

        # passage_tail：末題 (D) 選項之後的內容
        last_q_num = sub_qs[-1]['number']
        last_q_idx = find_q_start_index(content, last_q_num)
        last_q_body_raw = content[last_q_idx:]
        passage_start = find_passage_start(last_q_body_raw)
        passage_tail = last_q_body_raw[passage_start:].strip()

        passage = (passage_head + '\n' + passage_tail).strip()

        # --- 步驟 4：末題重新解析（乾淨範圍） ---
        last_q_clean_body = last_q_body_raw[:passage_start]
        last_q = extract_question(last_q_num, last_q_clean_body, '題組子題')
        sub_qs = [q for q in sub_qs if q['number'] != last_q_num]
        if last_q and len(last_q['options']) >= 3:
            sub_qs.append(last_q)
        sub_qs.sort(key=lambda x: x['number'])

        # --- 步驟 5：圖表歸屬 ---
        passage_figs = list(set(re.findall(
            r'[圖表]\([一二三四五六七八九十\d]+\)', passage
        )))
        passage_pages = resolve_figure_pages(passage_figs)

        for q in sub_qs:
            # 子題 figures 移掉 passage 已領的圖
            q['figures'] = [f for f in q['figures'] if f not in passage_figs]
            q['group_range'] = [q_start, q_end]
            q['type'] = '題組子題'
            q['group_id'] = group_id

        # --- 步驟 6：passage 寫入首題 ---
        first = sub_qs[0]
        first['passage'] = passage if passage else None
        first['passage_figures'] = passage_figs
        first['passage_figure_pages'] = passage_pages
        for other in sub_qs[1:]:
            other['passage'] = None
            other['passage_figures'] = []
            other['passage_figure_pages'] = {}

        questions += sub_qs

    return questions
```

### 4.4 `find_passage_start()` 啟發式

```python
def find_passage_start(last_q_body_raw: str) -> int:
    """
    在末題 body 中，找到 (D) 選項結束、passage 開始的位置。
    策略（由穩→鬆，多道防線）：

    L1: 找「(D)」標記 → 找 (D) 之後第一個連續換行後的完整段落起點
    L2: 找 (D) 選項內容的最後一個「句號」+ 換行 → passage 從下一行開始
    L3: fallback：從 (D) 之後用 regex 匹配「連續中文字符段落開頭」
    L4: 最終 fallback：從 body 端末倒數 200 字處開始（粗切）
    """
    # 找最後一個 (D) 位置
    d_matches = list(re.finditer(r'\(D\)', last_q_body_raw))
    if not d_matches:
        return len(last_q_body_raw)

    last_d = d_matches[-1]
    after_d = last_q_body_raw[last_d.end():]

    # L1: 連續兩個換行之後的段落
    m = re.search(r'\n\s*\n\s*(\S)', after_d)
    if m:
        # 驗證：這段是否是「正常中文段落」而非下一個 (A) 或題號
        peek = after_d[m.start():m.start() + 20]
        if not re.match(r'\([A-D]\)|\d{1,2}\.\s', peek.strip()):
            return last_d.end() + m.start()

    # L2: 找 (D) 內容的最後一個句號 + 換行
    m = re.search(r'[。](\s*\n)', after_d)
    if m:
        return last_d.end() + m.end()

    # L3: 找連續中文字符段落（啟發式）
    m = re.search(r'(?<=\n)\s*[\u4e00-\u9fff]{10,}', after_d)
    if m:
        return last_d.end() + m.start()

    # L4: fallback — 如果 (D) 後不足 200 字，直接取整段為 passage
    if len(after_d) < 200:
        return last_d.end()
    return len(last_q_body_raw)
```

> [!NOTE]
> 控制字元清理在 `parse_questions()` 前處理階段完成，因此 `find_passage_start` 處理的是已淨化文字。

### 4.5 `extract_question` 強化

```python
def extract_question(qnum: int, body: str, qtype: str) -> dict | None:
    """Extract stem and options from question body text."""

    # 控制字元二次清理（安全網）
    body = re.sub(r'[\x00-\x09\x0b-\x1f]', '', body)

    opt_a = body.find('(A)')
    if opt_a == -1:
        return None

    stem = body[:opt_a].strip()
    stem = re.sub(r'\s+', '', stem)  # 中文不需空格

    options_raw = body[opt_a:]

    # 改用結構邊界切選項
    opts = {}
    # 策略：每個 (X) 取到「下一個 \n(A-D)」、「\n題號. 」或字串結尾
    for letter in ['A', 'B', 'C', 'D']:
        if letter == 'D':
            pat = rf'\({letter}\)\s*(.+?)$'
        else:
            next_letter = chr(ord(letter) + 1)
            pat = rf'\({letter}\)\s*(.+?)(?=\n\({next_letter}\)|\n\d{{1,2}}\.\s|\Z)'
        m = re.search(pat, options_raw, re.DOTALL)
        if m:
            opts[letter] = re.sub(r'\s+', '', m.group(1).strip())

    # 如果少於 3 個選項，嘗試 fallback（原 regex）
    if len(opts) < 3:
        for letter in ['A', 'B', 'C', 'D']:
            if letter not in opts:
                m = re.search(rf'\({letter}\)\s*(.+?)(?=\([A-D]\)|\Z)', options_raw, re.DOTALL)
                if m:
                    opts[letter] = re.sub(r'\s+', '', m.group(1).strip())

    # 圖表引用（從子題 stem + 選項，不含 passage）
    all_text = stem + ''.join(opts.values())
    figs = list(set(re.findall(r'[圖表]\([一二三四五六七八九十\d]+\)', all_text)))

    return {
        'number': qnum,
        'type': qtype,
        'group_range': None,
        'stem': stem,
        'options': opts,
        'figures': figs,
    }
```

### 5.3 互動行為

| 功能 | 說明 |
|------|------|
| **passage 收合** | 預設展開；點擊「收合」按鈕 → 只顯示首行 + 「展開全文」；再點「展開」恢復 |
| **passage 固定** | 切換同 group 子題（52→53→54）時，passage DOM **不重新渲染**，只更新右欄題目 |
| **圖片區 sticky** | 左欄圖區 `position: sticky; top: 120px`，滾動時圖片保持可見 |
| **圖片區切換** | passage 圖常駐；當前子題有自有圖時疊加在左欄；切到無圖子題時移除子題圖 |
| **題組內切換** | 按鈕「上一題 / 下一題」僅在同 group 內循環；到邊界時按鈕 disabled |
| **題組指示器** | 「第 52-54 題 (1/3)」顯示當前 group 內位置 |
| **無圖退化** | passage 無圖（圖區內無內容）時，`group-question` 自動加 `.no-fig` class，stem 跨全寬 |
| **跨 group 導覽** | 51→52 切換到新 group 時重建 passage + 圖區（全量重新渲染） |
